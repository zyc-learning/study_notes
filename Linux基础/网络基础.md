

# 网络基础

## 零、概述

### 一、交换方式

#### 1.电路交换

工作流程：建立连接（分配通信资源） -> 通话（一直占用通信资源） -> 释放连接（归还通信资源）

重要特点：在通话的全部时间内，通话的两个用户始终占用端到端的通信资源



#### 2.分组交换

要发送的整块数据称为一个报文(message)，较长的报文不宜传输，这对交换结点的缓存容量有很大的需求。因此，需要将较长的报文划分为较小的数据段。在每一个数据段前面，加上必要的控制信息组成的首部(header)，构成一个分组(packet)，也称为包，分组的首部也可称为包头。

首部作用：包含分组的目的地址，用于转发分组。

工作流程：源主机将分组发送到分组交换网中，仅占用当前一条链路，分组交换网中的交换节点收到一个分组后，先缓存下来，从首部提取出目的地址，然后按照目的地址查找自己的路由表，找到对应的转发接口后，将分组转发出去，把分组交给下一个交换节点。经过多个交换节点转发后，分组最终被转发到目的主机。目的主机在接收到各分组后，去掉首部，将各数据段组合还原出原始报文。



各角色作用：

发送方：构造分组，发送分组

交换节点：缓存分组，转发分组

接收方：接受分组，还原报文



优点：无建立连接和释放连接的过程；分组传输过程中逐段占用通信链路，有较高的通信线路利用率；交换节点可以为每一个分组独立选择转发路由，使得网络有很好的生存性。

缺点：分组首部带来了额外的传输开销；交换节点存储转发分组会造成一定的时延；无法确保通信时端到端资源全部可用，在通信量较大时可能造成网络拥塞；分组可能会出现失序和丢失等问题。



#### 3.报文交换

报文交换是分组交换的前身。在报文交换中，报文被整个的发送，而不是拆分成若干分组进行发送。交换节点将报文整体接收完成后才能查找转发表，将整个保温转发到下一个节点。

因此，报文交换比分组交换带来的转发时延要长很多，需要交换节点具有的缓存空间也大很多。



#### 4.三种交换方式对比

连续传送大量的数据，并且数据传送时间远大于建立连接的时间，使用电路交换有较高的传输速率。

报文交换和分组交换都不需要建立连接（预先分配通信资源），在传送计算机的突发数据时可以提高线路的利用率。

将报文构造成若干个更小的分组进行分组交换，比将整个报文进行报文交换的时延更小，还可以避免太长的报文长时间占用链路，有利于差错控制，同时具有更好的灵活性。



### 二、计算机网络的定义和分类

#### 1.定义

计算机网络主要是由一些通用的、可编程的硬件互连而成的，而这些硬件并非专门用来实现某一特定目的，这些可编程的硬件，能够用来传送多种不同类型的数据，并能支持广泛和日益增长的应用。



#### 2.分类

##### 1.按交换方式分类

电路交换、报文交换和分组交换



##### 2.按使用者分类

1.公用网(Public Network)

电信公司出资建造的大型网络。所有愿意按电信公司的规定缴纳费用的人都可以使用。

2.专用网(Private Network)

某个部门为满足本单位的特殊业务工作的需要而建造的网络，这种网络不向本单位以外的人提供服务。



##### 3.按传输介质分类

1.有线网络

2.无线网络



##### 4.按覆盖范围分类

1.广域网WAN(Wide Area Network)：作用范围为几十到几十千米

2.城域网MAN(Metropolitan Area Network)：使用范围为五到五十千米

3.局域网LAN(Local Area Network)：作用范围为一千米左右

4.个人区域网PAN(Personal Area Network)：作用范围为十米左右



##### 5.按拓扑结构分类

总线型、星型、环型、网状型



### 三、计算机网络的性能指标

#### 1.速率

比特(bit，记为小写b)是计算机中数据量的基本单位，一比特就是二进制数字中的一个1或0。

数据量的常用单位有字节(byte，记为大写B)，千字节(KB)，兆字节(MB)，吉字节(GB)以及太字节(TB)、

换算方式：

| 数据量单位 | 换算关系       |
| ---------- | -------------- |
| b          | 基本单位       |
| B          | 1B=8b          |
| KB         | 1KB=2^10B      |
| MB         | 1MB=K*KB=2^20B |
| GB         | 1GB=K*MB=2^30B |
| TB         | 1TB=K*GB=2^40B |



速率是指数据的传输速率（每秒传送多少个比特），也成为数据率(Data Rate)，比特率（Bit Rate）

速率的基本单位是比特/秒（bit/s，可记为b/s，bps），常用单位有kb/s或kbps，Mb/s或Mbps，Gb/s或Gbps，Tb/s或Tbps。

换算方式：

| 速率单位 | 换算关系             |
| -------- | -------------------- |
| bps      | 基本单位             |
| kbps     | kbps=10^3bps         |
| Mbps     | Mbps=k*kbps=10^6bps  |
| Gbps     | Gbps=k*Mbps=10^9bps  |
| Tbps     | Tbps=k*Gbps=10^12bps |



#### 2.带宽

带宽在模拟信号系统中的意义：

某个信号所包含的各种不同频率成分所占据的频率范围，或称为频带宽度

单位：Hz(kHz，MHz，GHz)

带宽在计算机网络中的意义：

用来表示网络的通信线路所能传送数据的能力，即在单位时间内从网络中的某一点到另一点所能通过的最高数据率

单位：bps(kbps，Mbps，Gbps，Tbps)

以上两种表述方式的联系：线路频率带宽越宽，其所传输数据的最高数据率也越高

数据传输速率 = min [主机接口速率，线路带宽，交换机或路由器的接口速率]



#### 3.吞吐量

吞吐量是指在单位时间内通过某个网络或者接口的实际数据量，吞吐量常被用于对实际网络的测量，以便获知到底有多少数据量通过了网络。**吞吐量受网络带宽的限制**



#### 4.时延

时延是指数据从网络的一端传送到另一端所耗费的时间，也称为延迟或迟延。数据可由一个或者多个分组，甚至一个比特构成。

时延类型以及计算方式：

​	发送时延=分组长度(b)/发送速率(bps)

​	传播时延=信道长度(m)/信号传播速率(m/s)（自由空间：`3.0*10^8 m/s `铜线：`2.3*10^8 m/s` 光纤：`2.0*10^8 m/s`）

​	在路由器中的排队时延(不方便计算)

​	在路由器中的处理时延(不方便计算)

**不计算接收时延的原因：目的主机接收分组信号和分组信号在链路上传播是同时进行的，在总时延中包含接收时延，就把接收时延包含在内**

总时延公式 = (n+m)发送时延 + (n+1)传播时延 其中，n为路由器数量，m为分组个数。**使用前提：转发时延和发送时延一致**



#### 5.时延带宽积

时延带宽积时传播时延和带宽的乘积。表示管道可容纳的比特数量。当发送端连续发送数据时，再发送的第一个比特到达终点时，链路中充满了时延带宽积个比特。因此，链路时延带宽积也被称为以比特为单位的链路长度。



#### 6.往返时间RTT

往返时间RTT(Round-Trip Time)是指从发送端发送数据分组开始，到发送端收到接收端发来的相应确认分组为止，总共耗费的时间。



#### 7.利用率

利用率有信道利用率和网络利用率两种。

信道利用率是指某条信道有百分之几的时间是有数据通过的。完全空闲的信道利用率为零。

网络利用率是指全网络的信道利用率的加权平均值。

由排队论可知，当某条信道利用率增大时，该信道的引起的时延会迅速增加。当网络通信量较少时，时延不是很大；在网络通信量不断增大时，分组在交换节点中排队时延随之增大，因此网络时延引起的时延就会增大。

设D0为网络空闲时的时延，D为网络当前时延，U为网络现在的利用率，三者有如下关系：

D = D0/(1 - U)



#### 8.丢包率

丢包率指的是在一定时间范围内，传输过程中丢失的分组数量与总分组数量的比率。丢包率可以反映网络拥塞情况：

无拥塞时：丢包率为0；轻度拥塞时：丢包率为1-4%；严重拥塞时：丢包率为5-15%

丢包率分为网络丢包率，接口丢包率，节点丢包率，链路丢包率，路径丢包率

分组丢失主要情况有：

1.分组在传输过程中出现误码，被传输路径中的节点交换机或目的主机检测出误码而丢失

2.节点交换机根据丢弃策略主动丢弃分组，例如接受的分组已超过阈值，丢弃多余的分组。



### 四、常见网络体系结构

#### 1.OSI体系（法律上的国际标准）

物理层->数据链路层->网络层->运输层->会话层->表示层->应用层



#### 2.TCP/IP体系（事实上的国际标准）

网络接口层->网际层->运输层->应用层

IP协议是网际层核心协议，TCP和UDP是运输层的两个重要协议，应用层有许多协议：http,smtp,dns,rtp等

IP协议负责互连不同的网络接口，未各种网络应用提供服务



#### 3.原理体系结构

物理层->数据链路层->网络层->运输层->应用层

物理层关注点：采用什么介质进行传输，采用什么物理接口连接传输媒体，采用什么形式传输比特0和1。即解决使用何种信号来传输比特的问题。

数据链路层关注点：如何标识网络中的主机（主机编址问题）、目的主机如何从信号所表示的一连串比特流中区分出地址和数据。即解决分组在一个网络（或一段链路）上传输的问题。

网络层关注点：如何标识各网络以及网络中的各主机（网络和主机共同编址问题）、路由器如何转发分组，如何进行路由选择。即解决分组在多个网络上传输和路由的问题。

运输层关注点：如何解决进程之间基于网络的通信问题、出现传输错误，如何处理。即解决进程之间基于网络的通信问题。

应用层关注点：通过应用进程间的交互来完成特定的网络应用。



### 五、专用术语

#### 1.实体

实体指任何可发送或接收信息的硬件或软件进程。

对等实体：收发双方相同层次中的实体

协议数据单元PDU：对等层次之间传输的数据包称为该层的协议数据单元

服务数据单元SDU：同一系统内。层与层之间交换的数据包称为服务数据单元

多个SDU可以合成一个PDU，一个SDU也可以划分为几个PDU。



#### 2.协议

协议是控制两个对等实体进行逻辑通信的规则的集合

协议三要素：语法、语义、同步。

语法定义了通信双方所交换信息的格式

语义定义收发双方要完成的操作

同步定义了收发双方的时序关系



#### 3.服务

在协议控制下，两个对等实体间的逻辑通信使得本层能够向上一层提供服务。要实现本层协议，需要使用下面一层所提供的服务。协议是”水平的“，服务是”垂直的“。实体看得见相邻下层所提供的服务，但是不知道实现该服务的具体协议。

服务访问点：在同一系统中相邻两层的实体交换信息的逻辑接口，用于区分不同的服务类型。例如：数据链路层服务访问点是帧“类型”字段，网络层的服务访问点是IP数据报首部中的”协议“字段，运输层服务访问点是”端口号“

服务原语：上层使用下层所提供的服务必须通过与下层交换一些命令。



## 一、物理层

物理层要实现的功能：在各种传输媒体上传输比特0和1

物理接口特性：机械特性、电气特性、功能特性、过程特性



### 1.传输媒体

传输媒体是计算机网络设备之间的物理通路，也称为传输介质或传输媒介

传输媒体分为导向型传输媒体（固体媒体）和非导向型传输媒体（自由空间）。常见的导向型传输媒体有同轴电缆，双绞线，光纤。常见的非导向型传输媒体有无线电波，微波，红外线，大气激光，可见光。



### 2.传输方式

#### 1.并行传输和串行传输

并行传输：发送端和接收端之间只有一条数据传输路线。数据多个比特在传输路线上逐个依次传输。多用于远距离传输。

串行传输：发送端和接收端有多条传输线路，数据多个比特被分别安排在不同的数据传输路线上同时传输。通常仅用于短距离传输。



#### 2.同步传输和异步传输

同步传输：以比特为单位，数据块以比特流的形式传输，字节之间没有间隔，也没有起始位和终止位，接收端在每个比特信号的中间时刻进行采样，以判别接收到的是比特0还是比特1。就要求收发双方对表示比特的信号的时间长度达成一致，即为同步。在不采取任何其他措施的情况下，由于传输时会有接收方对信号采样时刻的误差积累，收发双方的时钟频率无法达到严格同步。

收发双方时钟同步的方法：外同步：在收发双方之间增加一条时钟信号线。发送端在发送数据信号的同时，发送一路时钟信号，接收端在时钟信号的指挥下，对数据信号进行采样，这样实现收发双方的同步。

内同步：发送端将时钟信号编码到发送数据中一起发送，例如曼彻斯特编码和差分曼彻斯特编码自含始终编码，具有自同步能力。

异步传输：以字节为传输单位，字节之间的时间间隔不固定，接收端只在每个字节的起始处对字节内的比特实现同步，因此一般要给每个字节添加起始位和结束位。

异步是指字节之间的异步，即字节之间的时间间隔不固定。但是字节内的每个比特仍要同步，即各比特的持续时间是相同的。



#### 3.单向通信、双向交替通信和双向同时通信

单向通信（单工）：指只能有一个方向的通信，没有双向的交互。（无线电广播，电视广播）

双向交替通信（半双工）：通信双方都可以发送信息和接收信息，但是对于任何一方发送信息和接收信息不能同时进行。（对讲机，总线型以太网的各主机）

双向同时通信（全双工）：通信双方都可以同时发送信息和接收信息。（电话，交换式以太网上的各主机）



### 3.编码与调制

#### 1.基本概念

基带信号包含较多的低频分量，甚至有直流分量，而许多信道并不能传输这种低频分量或直流分量，为解决这一问题，就必须对基带信号进行调制。

调制可分为两大类：

基带调制：仅对基带信号的波形进行变换，使它能够与信道特性相适应。由于是把数字信号转换为另一种形式的数字信号，也称为编码

带同调制：使用载波进行调制，把基带信号的频率范围搬移到较高的频段，并转换为模拟信号。



信号的编码单元称为码元，在使用时间域的波形表示信号时，代表不同离散数值的基本波形被称为码元

#### 2.常用编码方式

双极性不归零编码：需要给收发双方在添加一条时钟信号线，发送方通过数据信号线给接收方发送数据的同时，还通过时钟信号线给接收方发送时钟信号，接收方按照接收到的时钟信号的节拍，对数据信号线上的信号进行采样。对于计算机网络，宁愿利用这跟传输线传输数据信号，而不是传输时钟信号。编码效率高，但是存在同步问题

双极性归零编码：归零是指信号在每个码元期间会回归到零电平。在每个码元的中间时刻信号都会回归到零电平，接收方只要在信号归零后采样即可。然而，归零编码中大部分的数据带宽，都用来传输归零而浪费了。自同步信号，但编码效率低。

曼彻斯特编码：码元的中间时刻的电平跳变既表示时钟信号，也表示数据，可自定义正跳变表示1还是0，负跳变表示0还是1。自同步信号，10Mbps传统以太网

差分曼特斯特编码：在每个码元的中间时刻电平都会发生跳变。码元中间时刻的电平跳变仅表示时钟信号，而不表示数据，数据的表示在于每一个码元开始处是否有电平跳变，无跳变表示1，有跳变表示0.在传输大量连续1或连续0的情况下，差分曼彻斯特编码信号比曼彻斯特编码信号的变化少，在噪声环境下，检车有无跳变比检测跳变方向更不容易出错，因此差分曼彻斯特编码信号比曼彻斯特编码信号更易于检测。在传输介质接线错误导致高低电平翻转的情况下，差分曼彻斯特编码仍然有效。



#### 3.基本带通调制方法和混合调制方法

调幅（AM）：载波的振幅随基带数字信号而变化

调频（FM）：载波的频率随基带数字信号而变化

调相（PM）：载波的初始相位随基带数字信号而变化

混合调制方法

频率和相位：频率是相位随时间的变化率所以载波的频率和相位不能进行混合调制

相位和振幅：载波和相位可以结合起来进行调制，例如正交振幅调制QAM

QAM16有12个相位，每种相位有1或2种振幅可选，可调制出16中码元，每种码元可以对应表示4个比特(log216=4)



### 4.信道的极限容量

#### 1.造成失真的主要原因

在通信质量较好的信道上，会出现轻微失真，但是可是识别；在通信质量很差的信道上，会出现严重失真，无法识别的后果。

失真原因：

1.码元的传输速率：速率越高，信号经过传输后的失真就越严重

2.信号的传输距离：距离越远，信号经过传输后的失真就越严重

3.噪声干扰：噪声干扰越大，信号经过传输后的失真就越严重

4.传输媒体质量：传输媒体质量越差，信号经过传输后的失真就越严重

信道上传输的数字信号，可看作是多个频率的模拟信号进行多次叠加后形成的方波。如果数字信号中的高频分量在传输时受到衰减甚至不能通过信道，则接收到的波形前沿和后沿就变得不那么陡峭，每一个码元所占的时间界限也不再明确，这样，在接收端收到的信号波形就失去了码元的清晰界限，这种现象称为码间串扰。

如果信道的频带越宽，则通过的信号的高频分量就越多，那么码元的传输速率就可以更高，而不会导致码间串扰。然而信道的频率带宽有上限，不可能无限大，所以码元的传输速率也有上限。

奈氏准则：理想低通信道的最高码元传输速率=2W Baud ，其中 W是信道的频率带宽，单位为Hz；Baud是波特，即码元/秒。码元速率又称为波特率，调制速率，波形速率或符号速率。当1个码元只携带1比特的信息量时，波特率（码元/秒）与比特率（比特/秒）在数值上是相等的；当1个码元携带n比特的信息量时，波特率（码元/秒）转换成比特率（比特/秒）时，数值要乘以n

香农公式：C=Wlog2(1+S/N)其中：C：信道的极限信息传输速率(单位bps)，W:信道的频率带宽(单位Hz)，S：信道内所传信号的平均功率，N：信道内的高斯噪声功率。从香农公式可知：信道的带宽W或信道中的信噪比S/N越大，信息的极限传输速率C就越高

信噪比(dB)=10 lg (S/N)



### 5.信道复用技术

#### 1.基本原理

复用是指在一条传输媒体上同时传输多路用户的信号，当一条传输媒体的传输容量大于多条信道传输的总容量时，就可以通过复用技术，在这条传输媒体上建立多条通信信道，以便充分利用传输媒体的带宽。



#### 2.常见的信道复用技术

##### 1.频分复用FDM

将传输媒体的总频带划分成多个子频带，每个子频带作为一个通信子信道，每对用户使用其中的一个子信道进行通信，各子信道之间需要留出隔离频带，避免造成子信道之间的干扰。频分复用的所有用户同时占用不用的频带资源发送数据。



##### 2.时分复用TDM

每一个时分复用的用户在其相应时隙内，独占传输媒体的资源进行通信，时分复用的各用户所对应的时隙，构成了时分复用帧（TDM帧）。在使用时分复用技术进行通信的过程中，每个时分复用的用户所占用的时隙是周期性出现的，周期是TDM帧长。**注：**TDM帧是一段固定长度的时间，与数据链路层对等实体间逻辑通信的“帧”，是完全不同的概念。时分复用的所有用户在不同的时间占用同样的频带。



##### 3.波分复用WDM

即光的频分复用。在一根光纤上同时传输多个频率（波长）相近的光载波信号，实现基于光纤的频分复用技术。目前可在一根光纤上复用80路或者更多路的光载波信号，因此这种复用技术也称为密集波分复用技术DWDM。



##### 4.码分复用CDM

码分复用常称为码分多址（CDMA），它是在扩频通信技术的基础上发展起来的一种无线通信技术，与FDM和TDM不同，CDMA的每个用户可以在相同的时间使用相同的频带进行通信。这种系统发送的信号有很强的抗干扰能力，频谱类似白噪声。

CDMA将每个比特时间划分为m个更短的时间片，称为码片，m的取值通常为64或128，后续为简单起见，假设m取值为8。CDMA中的每个站点都被指派一个唯一的m比特码片序列：某个站点要发送比特1，则发送他自己的m比特码片序列；某个站点要发送比特0，则发送它自己的m比特码片序列的反码。

将码片序列中的比特0记为-1，比特1记为+1，可写出码片序列对应的码片向量。

如果有两个或多个站同时发送数据，则信道中的信号就是这些站各自所发送一系列码片序列或码片序列反码的叠加，为了从信道中分离出每个站的信号，给每个站指派码片序列时，必须遵从：

1.分配给每个站的码片序列必须各不相同，实际常采用伪随机码序列

2.分配给每个站的码片序列必须互相正交，即各码片序列相应的码片向量之间的规格化内积为0

例：站点A、B、C通过CDMA共享链路，A、B、C的码片序列分别是(1,1,1,1)、(1,-1,1,-,)和(1,1,-1,-1)。若C从链路上收到的序列是(2,0,2,0,0,-2,0,-2,0,2,0,2)，则C收到A发送的数据时多少

解：由于所给各站码片序列为4比特，因此将站点C收到的序列分成三部分，每部分由4比特组成：

(2,0,2,0,0,-2,0,-2,0,2,0,2)——>(2,0,2,0)，(0,-2,0,-2)，(0,2,0,2)

将站点A的码片序列(1,1,1,1)分别与上面三个部分进行规格化内积运算，根据结果可判断A发送的数据

(1,1,1,1)*(2,0,2,0) / 4=1，发送的是比特1

(1,1,1,1)*(0,-2,0,-2) / 4=-1，发送的是比特0

(1,1,1,1)*(0,2,0,2) / 4=1，发送的是比特1



## 二、数据链路层

链路：链路是指从一个节点到相邻节点的一段物理线路（有线或无线），而中间没有任何其他的交换节点。

数据链路：数据链路是基于链路的，当在一条链路上传输数据时，除需要链路本身，还需要一些必要的通信协议来控制这些数据的传输，把实现这些协议的硬件和软件加到链路上，就构成了数据链路。

计算机中的网络适配器（俗称网卡）和其他相应的软件驱动程序就实现了这些协议，一般的网络适配器包含了物理层和数据链路层这两层的功能。

帧：真事数据链路层对等实体之间在水平方向进行逻辑通信的协议数据单元PDU



数据链路层三个重要问题：封装成帧和透明传输，差错检测，可靠传输



### 1.封装成帧

封装成帧是指数据链路层给上层交付下来的协议数据单元PDU添加一个首部和一个尾部，使之成为帧。帧的首部和尾部中包含有一些重要的控制信息。例如以太网MAC帧首部包含：目的地址，源地址，类型三个字段，尾部仅包含：FCS字段（帧检验序列frame check sequence)。PPP帧首部包含：标志，地址，控制，协议四个字段，尾部包含：FCS，标志字段

帧首部和尾部作用之一就是帧定界。

为了提高数据链路层传输帧的效率，应当使帧的数据载荷长度尽可能地大于首部和尾部的长度。考虑到对缓存空间的需求以及差错控制等诸多因素，每一种数据链路层协议都规定了帧的数据载荷的长度上限，即最大传送单元(MTU)，例如以太网的MTU为1500字节。



### 2.透明传输

透明传输是指数据链路层对上层交付下来的协议数据单元PDU没有任何限制，就好像数据链路层不存在一样。例如，网络层交付给数据链路层的协议数据单元中含有和帧界定符一样的数值，对端在接收到这端数值时，会认为已经结束，后续不再接收，实际后续仍有数据，就不能实现透明传输。

**面向字节的物理链路使用字节填充的方法实现透明传输**：数据链路层在把帧交付给物理层之前，会对帧的数据载荷进行扫描，每出现一个帧定界符，就在其前面插入一个转义字符，接收方的数据链路层在物理层交付的比特流中提取帧，遇到第一个帧定界符时，认为这是帧的开始；当遇到转义字符时，就明白后面1字节的内容虽然与帧定界符相同，但它是数据，而不是帧定界符。剔除转义字符后，将其后面的内容作为数据继续提取，直到再次提取到帧定界符时，表明这是帧的结束。

同理，若帧数据载荷中有转义字符，在交付前会在转义字符前加上一个转义字符。

**面向比特的物理链路使用比特填充的方法实现透明传输**：数据载荷部分恰好包含作为帧定界符的特定位串，发送方的数据链路层在将该帧交付给物理层进行发送前，会扫描帧的数据载荷，在出现5个连续的比特1后面添加1个比特0，经过比特0填充后的数据载荷，就可以确保不会包含帧界定符。

接收方的数据链路层，在把数据载荷向上交付网络层之前，对数据载荷进行扫描，每当发现5个连续的比特1时，就把后面的1个比特0删除，以此还原出原始的数据载荷



### 3.差错检测

误码概念：比特在传输过程中可能会产生差错（称为比特差错或误码）：比特1可能变成比特0，比特0可能变成比特1。在一段时间内，传输错误的比特数量占所传输比特总数的比率称为误码率(BER)。提高链路的信噪比，可以降低误码率，但是在实际的通信链路上，不可能使误码率下降为0。

使用差错检测技术来检测数据在传输过程中是否产生了比特差错，是数据链路层索要解决的重要问题之一。

发送方的数据链路层采用某种检错技术，根据帧的内容计算出一个检错码，将检错码填入帧尾部。接收方的数据链路层，从帧尾部取出检错码，采用与发送方相同的检错技术，就可以通过检错码检错出帧在传输过程中是否出现了误码。帧尾部中用来存放检错码的字段称为帧检测序列FCS。



常用的检错方式

#### 1.奇偶校验

奇校验是在待发送的数据后面添加一个校验位，使得添加该校验位后的整个数据中比特1的个数是奇数

偶校验是在待发送的数据后面添加一个校验位，使得添加该校验位后的整个数据中比特1的个数是偶数

在所传输的数据中，如果有奇数个位发生误码，则所包含比特1的数量的奇偶性会发生改变，可以检验出误码；在所传输的数据中，如果有偶数个位发生误码，则所包含比特1的数量的奇偶性不会发生改变，无法检验出误码（漏检）。

奇校验：当在数据位产生了一位误码，"1"数量的奇性改变，可以检验出错误；当在校验位产生了一位误码，"1"数量的奇性改变，可以检验出错误；当在数据位和校验位产生了两位误码，"1"数量的奇性不改变，不能检验出错误

偶校验：当在数据位产生了一位误码，"1"数量的偶性改变，可以检验出错误；当在校验位产生了一位误码，"1"数量的偶性改变，可以检验出错误；当在数据位和校验位产生了两位误码，"1"数量的偶性不改变，不能检验出错误



#### 2.CRC循环冗余校验

漏检率极低的循环冗余校验检错技术（Cyclic Redundant Check）

基本思想：

```
收发双方约定好一个生成多项式G(x);
发送方基于待发送的数据和生成多项式G(X)，计算出差错检验码（冗余码），将冗余码添加到待发送数据的后面一起传输；
接收方在收到数据和冗余码后，通过生成多项式G(x)来计算收到的数据和冗余码是否产生了误码。
```

发送方CRC操作：
被除数：待发送的数据+生成多项式最高次个0

除数：生成多项式各项系数构成的比特串

余数：添加到待发送的数据后。余数长度应与生成多项式最高次数相同，作为冗余码。

商：仅作为标记，防止计算过程对错位

除法规则：二进制的模二除法（相当于异或计算）

接收方CRC操作：
被除数：接收到的数据和冗余码

除数：生成多项式各项系数构成的比特串

余数：若为0，可判断接收的数据和冗余码中无误码；若不为0，判断出数据或冗余码中出现了误码



生成多项式例子：

G(x)=`x^4+x^2+x+1=1*x^4+0*x^3+1*x^2+1*x+1*x^0`

生成多项式个系数构成的比特串：10111

常用的生成多项式：

```
CRC-16:x^16+x^15+x^2+1
CRC-CCITT:x^16+x^12+x^5+1
CRC-32:x^32+x^26+x^23+x^22+x^16+x^12+x^11+x^10+x^8+x^7+x^5+x^4+x^2+x+1
```



例题：

待发送数据：101001，生成多项式G(x)=`x^3+x^2+1`，计算冗余码

步骤

```
1.构造被除数
101001000
2.构造除数
1101
3.二进制模二除法
得到余数1
4.检查余数
位数不足3位，补成001
```



例题：

接收到的信息是101101001，生成多项式为G(x)=`x^3+x^2+1`，判断传输是否误码

步骤

```
1.构造被除数
101101001
2.构造除数
1101
3.二进制模二除法
得到余数11，传输过程中出现了误码
```



奇偶校验、循环冗余校验，只能检测出传输过程中出现了差错，但不能定位错误，因此无法纠正错误。若想纠正传输中的差错，可以使用冗余信息更多的纠错码（例如海明码）进行前向纠错，但纠错码开销较大，在计算机网络中较少使用。在计算机网络中，通常采用检错重传方式来纠正传输中的差错，或者仅仅丢弃检测到差错的帧，取决于数据链路层向其上层提供的是可靠传输服务还是不可靠传输服务。



### 4.可靠传输

#### 1.概念

在处理误码时，由数据链路层向上提供的服务类型决定实现可靠传输的方法：

不可靠传输服务： 仅仅丢失有误码的帧，其他什么也不做

可靠传输： 通过某种机制实现发送方发送什么，接收方最终就能收到什么

在一般情况下，有线链路的误码率比较低。为了减小开销，并不要求数据链路层向其上层提供可靠传输服务。即使出现了误码，可靠传输的问题由其上层处理。无线链路易受干扰，误码率比较高，因此要求数据链路层必须向上层提供可靠传输服务。

传输差错有：误码（比特差错）、分组丢失、分组失序、分组重复。误码出现在数据链路层及其下层，分组丢失、分组失序、分组重复出现在数据链路层的上层。



实现可靠传输的方式

#### 2.停止等待协议（Stop-and-Wait,SW）

特点：有确认、否认和重传 超时重传 分组编号机制

##### 1.实现原理

```
发送方给接收方发送一个数据分组；
接收方接受后，对其进行差错检测，没有检测出误码，则接受该数据分组，并给发送方发送确认分组(ACK)；
发送方收到对所发送数据分组的确认分组后，才能发送下一个数据分组；

若发送方在发送时出现了一个分组出现了误码；
接收方收到后，对其进行差错检测，检测出误码，丢弃该数据分组，并给发送方发送否认分组(NAK)；
发送方收到对所发送数据分组的否认分组后，知道了之前发送的数据分组出现了误码而被接收方丢弃，于是立刻重传该数据分组；
接收方接受后，对其进行差错检测，没有检测出误码，则接受该数据分组，并给发送方发送确认分组(ACK)；
发送方收到对所发送数据分组的确认分组后，才能发送下一个数据分组；

## 因此，发送方每发送完一个数据分组后，并不能立刻将该数据分组从发送缓存中删除，因为之后有可能需要重传该数据分组。只有在收到针对该数据分组的确认分组后，才能将其从发送缓存中删除。

超时重传：当数据分组在传输过程中丢失时，接收方不会发送相应的ACK和NAK，发送方会一直等待接收ACK或NAK。为解决这个问题，发送方在每发送完一个数据分组就启动一个超时计时器(Timeout Timer),若到了超时计时器所设置的超时重传时间(Retransmission Time-Out,RTO)，但发送方仍未收到接收方的ACK或NAK，就重传之前已发送过的数据分组。
记收发双方之间的平均往返时间为RTT，一般可将超时重传时间RTO设置为略大于收发双方的平均往返时间RTT

当数据分组成功接收，接收方ACK丢失，发送方重传数据分组，导致分组重复。为避免分组重复的传输错误，必须给每个分组带上序号。对于停等协议，由于每发送一个数据分组就停止等待，只要保证每发送一个新的数据分组，其序号与上次发送的数据分组不同就可以了，因此用一个比特来编号就够了，序号有0和1这两个。同时确认分组也需要进行编号
在识别到分组重复这种传输差错，接收方丢弃重复的数据分组，然后向发送方再发送一个确认分组
```

**注意事项**：

```
1.使用超时重传机制后，就可以不使用否认机制了，这样可使协议实现起来更加简单，但是，如果点对点链路的误码率较高，使用否认机制可以使发送方在超时计时器超时前就尽快重传。
2.为了让接收方能够判断所收到的数据分组是否重复，需要给数据分组编号。由于停等协议的特性，只需1个比特编号即可，即序号0和序号1。
3.为了让发送方能够判断所受到的确认分组是否重复的，需要给确认分组编号，所用比特数量与数据分组所用比特数量一样。数据链路层一般不会出现确认分组迟到的情况，因此在数据链路层实现停等协议可以不用给确认分组编号。
4.给超时计时器设置的超时重传时间RTO应当仔细选择，一般将RTO设置为略大于收发双方的平均时间RTT。
在数据链路层，点对点的往返时间RTT比较固定，RTO就比较好设定。
在运输层，由于端到端往返时间非常不确定，设置合适的超时重传时间RTO有时并不容易。
5.停等协议数据自动请求重传(Automatic Repeat reQuest,ARQ)协议，即重传的请求是发送方自动进行的，而不是接收方请求发送方重传某个误码的数据分组。
```



##### 2.停等协议的信道利用率

记发送方发送数据分组所耗费的发送时延为TD，信号在收发双方之间往返传播所耗费的时间为RTT ，接收方发送确认分组所耗费的发送时延为TA

信道利用率U=TD/(TD+RTT+TA),TA<<TD  <=> U=TD/(TD+RTT) 当 RTT >> TD 时，例如卫星链路RTT很大，U很低；当 RTT << TD时，例如无线局域网RTT远小于TD，U比较高。

**若出现超时重传，对于传送有用的数据信息来说，信道利用率还要降低**

在往返时间RTT相对较大的情况下，为了提高信道利用率，收发双方不适合采用停等协议，而是可以选择回退N帧协议或选择重传协议。



#### 3.回退N帧协议（Go-back-N,GBN）

流水线传输方式：连续发送多个数据分组，而不必每发送完一个数据分组就停下来等待接收方的确认分组。这种方法对比停等协议可以显著提高信道利用率。

在使用流水线传输方式时，发送方不能无限制的连续发送数据分组，否则可能会导致网络中的路由器或接收方来不及处理这些数据分组，从而导致数据分组的丢失，这实际上是对网络资源的浪费，因此，必须采取措施来限制发送方连续发送数据分组的数量。

##### 1.回退N帧协议

回退N帧协议采用流水线传输方式，并且利用发送窗口来限制发送方连续发送数据分组的数量，这属于连续ARQ协议。

```
1.采用n个比特给分组编序号，序号范围是0~(2^n - 1)。

2.发送方需要维护一个发送窗口Wt，在未收到接收方确认分组的情况下，发送方可将序号落入Wt内的所有数据分组连续发送出去。采用n个比特给分组编序号，则Wt的取值范围是1<Wt<(2^n - 1)。如果将发送窗口的取值为1，则变成停止等待协议；如果发送窗口的值超过取值范围的上限，则会出现严重的错误。序号落在发送窗口的几个数据分组，可以连续发送，而序号落在发送窗口外的数据分组不允许发送。

3.接收方需要维护一个接收窗口Wr,只有正确到达接收方(无误码)，且序号落入Wr内的数据分组才被接收方接收。接收窗口Wr的取值只能是1，与停止等待协议一致。序号落在接收窗口的这个数据分组允许接收，而序号落在发送窗口外的数据分组不允许接收。

4.接收方每正确收到一个序号落入接收窗口的数据分组，就将接收窗口向前滑动一个位置，这样就有一个新的序号落入接收窗口。与此同时，接收方还要给发送方发送针对该数据分组的确认分组；发送方每收到一个按需确认的确认分组，就将发送窗口向前滑动一个位置，这样就有一个新的序号落入发送窗口，序号落入发送窗口内的数据可继续被发送。

5.在回退N帧协议的工作过程中，发送方的发送窗口和接收方的接收窗口按上述规则不断向前滑动。因此，这类协议又被称为滑动窗口协议。

回退N帧协议的接收方采用累计确认方法：接收方不必对收到的每一个数据分组都发送一个确认分组，而是可以在收到几个序号连续的数据分组后，对按序到达的最后一个数据分组发送确认分组。接收方何时发送累计确认分组，由具体实现决定。确认分组ACKn表明序号n及之前的所有数据分组都已正确接收。

优点：减少像网络中注入确认分组的数量；即使前面确认分组丢失，后面的确认分组无误，不必重传前面的数据。
缺点：不能向发送方及时准确的反映出接收方已正确的接受的所有数据分组的数量；前面的确认分组未丢失，后面的确认分组丢失，
```



##### 2.工作过程

无传输差错的情况

```
	发送方将序号落入发送窗口内的数据分组依次连续发送出去，经过因特网的传输正确到达了接收方。接收方按序接收它们，每接受一个，接收窗口就向前滑动一个位置，并给发送方发送针对所接受数据分组的确认分组。确认分组经过因特网的传输，正确到达了发送方，发送方每按序接受一个确认分组，发送窗口就像前滑动一个位置，这样就有新的序号落入到发送窗口。发送方可以将收到确认的数据分组，从发送缓存中删除了，接收方从接收缓存中及时取走已正确接收到的数据分组。
```

​	在无传输差错的情况下，回退N帧协议的信道利用率比停止等待协议的信道利用率有显著提高，提高的程度取决于发送窗口的大小。



超时重传、回退N帧的情况

```
	发送方将序号落入发送窗口内的数据分组依次连续发送出去，经过因特网的传输按序到达了接收方。2号数据分组在传输过程中产生了误码，接收方按序接收0号和1号数据分组，接收窗口向前滑动了两个位置，并给发送方发送0号和1号的确认分组，接收方通过2号数据分组中的检错码，检测出了2号数据分组中有误码，因此将其丢弃，接收窗口不向前滑动；由于后续数据分组的序号未落入接收窗口，因此接收方将它们丢弃，接收窗口不向前滑动。
	接收方每收到一个未按序到达的数据分组，也就是序号为落入接收窗口的数据分组，除了将它们丢弃外，还给发送方发送针对最近已按序接收的数据分组的确认分组。而接收方丢弃了无法按序接收的3号和4号数据分组，因此，接收方会发送两个针对1号数据分组的确认分组（总共三个1号数据分组的确认分组，一个是接收到返回的确认分组，另外两个是丢弃了3和4号返回的最近已接收到的数据分组即1号分组的确认分组）。
	发送方收到0号和1号确认分组，就将发送窗口向前滑动两个位置，这样就有新的序号落入发送窗口，发送方将新的数据分组依次发送出去。发送方在收到针对1号数据分组的确认分组后，发送方就知道了接收方并未按序正确接收2号数据分组，而有两个序号为落入接收窗口的数据分组，也不能接收而被丢弃。接收方在收到后续的5号和6号数据分组，由于它们的序号未落入接收窗口，因此，接收方将它们丢弃，接收窗口不向前滑动并返回给发送方两个针对1号数据分组的确认分组。
	发送方接收到了接收方针对1号数据分组的两个确认分组，此时发送方已收到针对1号数据分组的四个重复确认，发送方就知道了接收方并未按序正确接收2号数据分组，发送方此时可以立刻重传2号数据分组，也就是在2号数据分组的重传计时器超时前就提前进行重传。
```

​	尽管3号数据分组往后的数据分组到接收方时并未出现误码，但是接收方只能接收序号落入接收窗口内的数据分组。因此，一旦2号数据分组出现误码被接收方丢弃，其后连续发送的数据分组都要被重传。即一旦出错，就需要退回，重传已发送过的N个数据分组。

​	每当一个数据分组出现误码时就可能引起大量数据分组的重传，在信道质量较差（易出现误码）的情况下，回退N帧协议的信道利用率并不比停止等待协议的信道利用率高。



#### 4.选择重传（Selective Repeat,SR）

```
特性：用n个比特给分组编号，1<Wr<=Wt,当Wr超过Wt没有意义，1<Wr<=2^(n-1),当接收窗口Wr取最大值2^(n-1)时，Wt能取到的最大值也为2^(n-1)。
Wt+Wr<=2^n，以此来确保接收窗口向前滑动后，落入接收窗口内的新序号与之前的旧序号没有重叠，避免无法分辨新旧数据分组。

为了使发送方仅重传出现差错的数据分组，接收方不再采用累计确认，而需要对每一个正确接收的数据分组进行逐一确认。
```

```
1.假设采用3个比特给分组编序号，序号范围是0-(2^3-1),接收窗口Wr取最大值为2^(3-1),即Wr=4；发送窗口Wt能取到的最大值也是2^(3-1)，即Wt=4。
2.发送方将序号落入发送窗口内的0-3号数据分组连续发送给接收方，假设0号数据分组在传输过程中出现误码，接收方将其丢弃，接收方将失序，但正确到达接收方且序号落入接收窗口内的1-3号数据分组进行缓存，并给发送方发送相应的1-3号确认分组，缓存的1-3号数据分组暂不交付给上层，由于还未收到序号落入接收窗口内的数据分组，因此接收窗口不向前滑动。发送方收到失序的1-3号确认分组后，并不向前滑动发送窗口，但记录1-3号数据分组已被确认。(失序是指未收到0号确认分组，就受到了1-3号确认分组)因此，只有0号数据分组被超时重传，0号数据分组正确到达接收方后，接收方正确接收序号落入接收窗口的0号数据分组，并给发送方发送0号确认分组。接收方现在将接收窗口向前滑动4个位置，这样4-7落入接收窗口内。0号确认分组正确到达发送方后，发送方在收到0号确认分组后，将发送窗口向前滑动4个位置，这样4-7落入发送窗口内，此时发送方可以将4-7号数据分组发送出去，往后依次类推。
3.在后一组发送的时候，发送方可将发送缓存中序号已被移出发送窗口的数据分组删除了，接收方应尽快将接收缓存中序号已被yi'chu
```



### 5.点对点协议PPP

#### 概述

点对点协议(Point-to-Point Protocol，PPP)是目前使用最广泛的点对点数据链路层协议，点对点协议是因特网工程任务组于1982年制定的，经过多次修订，目前PPP已成为因特网的正式标准[RFC1661，RFC1662].

点对点协议主要应用：

1.因特网用户通过点对点链路连接到ISP(互联网服务提供商)进而连接到因特网。用户与ISP通信时，采用的数据链路层一般就是PPP协议。在以太网上运行的PPP协议是PPoE，使得ISP可以通过数字用户线路，电路调制解调器以及以太网等宽带接入技术，以以太网接口的形式为用户提供接入服务

2.点对点协议广泛应用于广域网路由器之间的专用线路。



从网络体系结构的角度看点对点协议PPP的组成

PPP是数据链路层协议，它将上层(网络层)交付下来的协议数据单元PDU封装成帧，为了支持不同的网络层协议，PPP协议包含了一套网络控制协议NCPs，链路控制协议LCP用于建立、配置以及测试数据链路的连接。PPP协议能够在多种类型的点对点链路上运行，如：面向字节的异步链路，面向比特的同步链路。



#### PPP协议的帧格式

```
              帧首部                  帧的数据载荷                        帧尾部
       标志  地址  控制   协议     封装上层的协议数据但与PDU         帧校验序列    标志
长度：  1B    1B   1B     2B      <=最大传输单元MTU=1500B             2B         1B

标志字段：是PPP帧的定界符，取值为0x7E
地址字段：取值为0xFF，预留（目前没有什么作用）
控制字段：取值为0x03，预留（目前没有shen'me
协议字段：其值用来指明帧的数据载荷应向上交付给哪个协议处理
帧检验序列字段：其值是使用循环冗余校验CRC计算出的检错码
```



#### PPP帧的透明传输

若帧的数据载荷中切好含有帧定界符，在不采取措施的情况下，会造成接收方对帧定界的误判。

帧首部和帧尾部定界符的取值从字节角度看取值为0x7E，从比特角度看取值为011111110。



##### 面向字节的异步链路时使用字节填充来实现透明传输

此时将帧首部和尾部中的帧定界符看作是取值为0x7E的一个字符。

发送方的处理：

将数据载荷中出现的每一个0x7E减去0x20（相当于异或0x20），然后在其前面插入转义字符0x7D。

若数据载荷中原来就含有0x7D，则把每一个0x7D减去0x20，然后在其前面插入转义字符0x7D。

将数据载荷中出现的每一个ASCII码控制字符（即数值小于0x20的字符），加上0x20（相当于异或0x20，将其转换成非控制字符），然后在其前面插入转义字符0x7D。

接收方的处理：

进行与发送方相反的变换，就可以正确地恢复出未经过字节填充的原始数据载荷



##### 面向比特的同步链路时使用零比特填充来实现透明传输

此时将帧首部和尾部中的帧定界符看作是取值为01111110的八比特特定位串。

发送方的处理：对帧的数据载荷进行扫描（一般由硬件完成），每出现五个连续的比特1时，则在其后填充一个比特0。

接收方的处理：对帧的数据载荷进行扫描，每出现五个连续的比特1时，就把其后的一个比特0删除。



#### PPP帧的差错检测

在帧尾部中包含有一个2B的帧序列检验序列FCS字段，其值是使用循环冗余校验CRC计算出的检错码。生成多项式为CRC-CCITT=X^16+X^12+X^5+1。计算范围是第一个标志位到帧校验序列位之间的部分（地址 控制 协议 封装上层的协议数据单元PDU）。

接收方每收到一个PPP帧，就进行CRC校验，若CRC检验正确，就收下这个帧；否则，就丢弃这个帧。使用PPP的数据链路层，向上提供的是不可靠数据传输服务。



#### PPP的工作状态

以用户主机拨号接入因特网提供者ISP的拨号服务器的过程为例

```
PPP链路的开始和结束状态都是静止状态，用户PC与ISP的拨号服务器之间并不存在物理层的连接。当检测到调制解调器的载波信号并建立物理层连接后，PPP进入链路的建立状态。

在建立状态下，链路控制协议LCP开始协商一些配置选项，若协商成功，进入鉴别状态；若协商失败，退回到静止状态。协商的配置选项包括最大帧长，鉴别协议等。可以不使用鉴别，也可以使用口令鉴别协议PAP或挑战握手鉴别协议CHAP，若通信双方无需鉴别或鉴别身份成功，进入网络状态；若鉴别失败，进入终止状态。

在进入网络状态后，PPP链路的两端通过互相交换网络层特定的NCP分组，来进行NCP(Nwrwork Control Protocol，网络控制协议)配置，如果PPP链路的上层使用的是IP协议，则使用IP控制协议IPCP来对PPP链路的每一端配置IP模块，例如分配IP地址。NCP配置完成后就进入打开状态。只要链路处于打开状态，双方就可以进行数据通信。

当出现故障或者链路的一端发出终止请求时，进入终止状态。当载波停止后就回到静止状态。
```



### 6.共享式以太网

以太网(Ethernet)以曾经被假想的电磁波传播介质——以太(Ether)命名。以太网最初采用无源电缆(不包含电源线)作为共享总线来传输帧，属于基带总线局域网，传输速率为2.94Mb/s。1980年出台通用的以太网标准（以太网标准的第一个版本DIX Ethernet V1），传输速率为10Mb/s。1982年出台DIX Ethernet V2，是第一个局域网产品的标准。1983年IEEE 802委员会的802.3工作组在DIX Ethernet V2的基础上制定出了IEEE的以太网标准，即IEEE 802.3，但是并没有被广泛使用。

以太网目前已经从传统的共享式以太网发展到交换式以太网，传输速率已经从10Mb/s提高到100Mb/s、1Gb/s甚至10Gb/s

A                       B                        C                             D
```

位于共享总线两端的两个站点之间的单程端到端的传播时延记为t(tao)。在t=0时刻主机A要发送帧，当检测到总线空闲96比特时间后，立即发送帧。在t=tao-delta(delta是D开始发送的时刻)时刻，站点A发送的帧未到达站点D，而站点D也要发送帧。尽管总线已被站点A占用，但是站点A发送的信号还未到达站点D，因此，站点D检测不到总线忙，站点D检测到总线空闲96比特时间后，立即发送帧，这必然会产生碰撞，碰撞时刻为t=tao-(delta/2)()，但此时双方都检测不到碰撞，碰撞信号沿总线传播到总线的两端。在t=tao时刻，站点D首先检测到碰撞，于是立即停止发送帧。在t=2tao-delta时刻，站点A也检测到了碰撞，于是立即停止发送帧。站点A和D发送帧都失败了，各自退避一段随机时间后再重新发送。

站点从发送帧开始，最多经过2tao(即delta->0)就可以检测出所发送的帧是否遭遇了碰撞。共享总线以太网的端到端往返时间2tao被称为争用期或碰撞窗口。站点从发送帧开始，经过争用期2tao这段时间还没有检测到配装，就可以肯定这次发送不会产生碰撞。

从争用期的概念可以看出，共享总线上的每一个站点从发送帧开始，到之后的一小段时间内，都有可能遭遇碰撞，而这一小段时间的长短是不确定的，它取决于另一个发送帧的站点与本站点的距离，但不会超过总线的端到端往返传播时延，即一个争用期2tao。显然，总线的长度越长(单程端到端传播时延越大)，网络中站点数量越多，发生碰撞的概率就越大。因此共享以太网的总线不能太长，接入的站点数量也不能太多。

10Mbps共享总线以太网(传统以太网)规定：争用期2tao的值为512比特的发送时间，即51.2us



#### 共享式以太网的最小帧长和最大帧长

```
----------------------------------------总线-------------------------------------
A                       B                        C                             D
```

站点A给D发送一个很短的帧，边发送边检测碰撞。由于该帧很短，很快就发送完毕，站点A不再检测碰撞。假设该帧在传送过程中，站点C也要发送帧，站点C在检测到信道空闲96比特时间后发送帧，这必然会产生碰撞。站点D最终收到遭遇碰撞的帧，丢弃即可。然而站点A并不知道已发送完毕的帧遭遇了碰撞，不会重发该帧。这样站点D就无法正确收到该帧。

为了确保共享总线以太网上的每一个站点在发送完一个完整的帧之前，能够检测出是否产生了碰撞，帧的发送时延就不能少于共享总线以太网端到端的往返时间，即一个争用期2tao。

10Mbps共享总线以太网，其争用期2tao的值规定为51.2us，因此其最小帧长为512b即64B。当某个站带你在发送帧时，如果帧的前64B没有遭遇碰撞，那么帧的后续部分也就不会遭遇碰撞。也就是说，遭遇碰撞，就一定是在帧的前64B之内。由于发送帧的站点边发送边检测碰撞，一旦检测到碰撞就立即终止帧的发送，此时已发送的数据量一定小于64B，因此，接受站点收到长度小于64B的帧，就可判断这是一个遭遇了碰撞而异常种植的无效帧，将其丢弃即可。



一般来说，帧的数据载荷的长度应远大于帧首部和尾部的总长度，这样可以提高帧的传输效率。如果不限制数据载荷的长度上限，这会带来一些问题。

```
----------------------------------------总线-------------------------------------
A                       B                        C                             D
```

站点A给D发送一个很长的帧，这会使得站点A长时间占用总线，而总线上的其他站点一直拿不到总线的使用权。同时由于帧很长，可能导致站点D的接收缓冲区无法装下该帧而产生溢出。

以太网V2的MAC帧的数据载荷的最大长度被限制为1500B，加上首部和尾部，以太网V2的MAC帧的最大长度为1518B；数据载荷的最小长度规定为46B，以满足最小帧长为64B的要求



#### 共享以太网退避算法

共享总线以太网中的各站点采用截断二进制指数退避算法来选择退避的随机时间。

截断二进制退避算法：

退避时间=基本退避时间*随机数r，其中基本退避时间为争用期2tao；r从离散的整数集合{0,1,...,(2^k - 1)中随机选出一个数，k=min{重传次数，10}}

如果连续多次发生碰撞，就表明可能有较多的站点参与竞争信道，但使用上述退避算法可使重传推迟的平均时间随重传次数而增大(即动态退避)，因而减小产生碰撞的概率。当重传达16次仍不能成功时，就表明同时打算发送帧的站点太多，以至于连续产生碰撞，此时应放弃重传并向高层报告。



#### 共享以太网信道利用率

假设总线上某站点可能产生多次碰撞并进行多次退避后，才成功发送一个帧。记帧的发送时延为T0。在最极端的情况下，源站点在总线的一端，目的站点在总线的另一端，因此要经过一个单程的端到端的传播时延tao后，总线才能完全进入空闲状态。

发送一帧所需的平均时间，由多个争用期、帧的发送时延以及单程端到端的传播时延tao，这三部分构成。

考虑一下理想情况：总线一旦空闲就有某个站点立即发送帧；各站点发送帧都不会产生碰撞；发送一帧占用总线的时间是(T0+tao)，帧本身的发送时间是T0。

因此，在理想状态下不考虑退避，极限信道利用率Smax=T0/(T0+tao)。为提高信道利用率，共享总线以太网端到端的距离不应太长，真的长度应尽量大。



#### 在物理层扩展以太网

##### 扩展站点与集线器之间的距离

共享总线以太网中两个站点之间的库里不能太远，否则它们之间所传输的信号就会衰减到使CSMA/CD协议无法正常工作。早期广泛使用粗同轴电缆或细同轴电缆共享总线以太网时，为了提高网络地理覆盖范围，常用的是工作在物理层的转发器。IEEE802.3标准规定，两个网段可用一个转发器连接起来，任意两个站点之间最多可以经过三个网段。

```
共享总线以太网网段1-----转发器------共享总线以太网网段2------转发器-------共享总线以太网网段3
```

随着使用双绞线和集线器的10BASE-T星型以太网成为以太网的主流类型，扩展网络覆盖就很少使用转发器了。10BASE-T星型以太网每个站点到集线器的距离不能超过100m，因此两站点间的通信距离最大不能超过200m。

在10BASE-T星型以太网中，可使用光纤和一对光纤调制解调器来扩展站点与集线器之间的距离。这种扩展方法较简单，所需付出的代价是：为站点和集线器各增加一个用于电信号和光信号转换的光纤调制解调器，以及它们之间的一对通信光纤。

信号在光纤中的衰减和失真很小，因此使用这种方法可以很简单地将站点与集线器之间地距离扩展至1000m以上，在生活中常见于光猫。



##### 扩展共享式以太网地覆盖范围和站点数量

以太网集线器一般具有8~32个接口，如果要连接的站点数量超过了单个集线器能够提供的接口数量，需要使用多个集线器，这样就可以连接成覆盖更大范围、连接更多站点的多级星型以太网。

采用多个集线器连接而成的多级星型以太网，在扩展了网络覆盖和站点数量的同时，也带来了一些负面因素。在物理层扩展的共享式以太网仍然是一个冲突域，不能连接太多的站点，否则可能会出现大量的碰撞，导致平均吞吐量太低。



#### 在数据链路层扩展以太网

##### 使用网桥在数据链路层扩展以太网

为了避免形成更大的冲突域，可以使用网桥在数据链路层扩展共享式以太网。网桥工作在数据链路层(包括其下的物理层)，而集线器仅工作在物理层。因此网桥具备属于数据链路层范畴的相关功能(网桥可以识别帧的结构；网桥可以根据帧首部中的目的MAC地址和网桥自身的帧转发表来转发或丢弃收到的帧)



##### 网桥主要结构以及基本工作原理

```
使用网桥将两个使用集线器的共享式以太网互连起来，形成一个覆盖范围更大、站点数量更多的以太网。
原来的两个共享式以太网，分别称为这个更大的以太网的一个网段，并且各自是一个独立的冲突域。
          主机B                        主机D
主机A     集线器         网桥           集线器     主机E
          主机C                        主机F
网桥结构：              转发表
      接口管理软件----网桥协议实体
               接口1----缓存----接口2
其中转发表是网桥转发帧的依据，里面记录有网桥所知道的网络中各主机的MAC地址与自己的各接口的对应关系。
网桥收到帧后，会在自身的转发表中查找帧的目的MAC地址，根据查找结构来转发或丢弃帧。
```



##### 透明网桥的自学习和转发帧的流程

透明网桥(Transparent Bridge)通过自学习算法建立转发表。透明是指以太网中各站点并不知道自己所发送的帧将会经过哪些网桥的转发，最终到达目的站点，也就是说，以太网的各网桥对于各站点而言是看不见的。

透明网桥的标准是IEEE802.1D，它通过一种自学习算法基于以太网中各站点之间的相互通信逐步建立起自己的转发表

```
网桥收到帧后进行登记(即自学习)，登记的内容为帧的源MAC地址和进入网桥的接口号
网桥根据帧的目的MAC地址和网桥的转发表对帧进行转发，包含以下三种情况：
	明确转发：网桥知道应当从哪个接口转发帧
	盲目转发：网桥不知道应当从哪个接口转发帧，只能将其通过除进入网桥的接口的其他所有接口转发。
	丢弃：网桥知道不应该转发该帧，将其丢弃。
注：如果网桥收到有误码的帧则直接丢弃；如果网桥收到一个无误码的广播帧，则不用进行查表，而是直接从除接收该广播帧的接口的其他接口转发该广播帧；转发表中的每条记录都有其有效时间，到期自动删除。这是因为各站点的MAC地址与网桥接口的对应关系并不是永久性的，例如某个站点更换了网卡，其MAC地址就会改变。
```



##### 透明网桥的生成树协议STP

为了提高以太网的可靠性，有时需要在两个以太网之间使用多个透明网桥来提供冗余链路。在增加冗余链路提高以太网可靠性的同时，却给网络；引入了环路。这样可能会使广播帧在环路中永久兜圈，造成广播帧充斥整个网络，网络资源被白白浪费，而网络中的主机之间无法正常通信。同时：

若两个网桥的转发表中都没有待转发单播帧目的MAC地址的相关记录，则该单播帧也会引起类似的情况；为简单起见，未考虑信号在总线上碰撞的情况。

为了避免广播帧在环路中永久兜圈，透明网桥使用生成树协议(Spanning Tree Protocol,STP)，可以增加冗余链路提高网络可靠性的同时，又避免环路带来的问题。不管网桥之间连接成了怎样复杂的带环拓扑，网桥之间通过交互网桥协议单元(Bridge Protocol Data Unit,BPDU)，找出原网络拓扑的一个连通子集(即生成树)，在这个子集里整个连通的网络中不存在环路。当首次连接网桥或网络拓扑发生变化时(人为改变或出现故障)，网桥都会构造生成树，以确保网络的连通。



### 7.网络适配器(网卡)和MAC地址

#### 网络适配器

要将计算机连接到以太网，需要使用相应的网络适配器(Adapter)，网络适配器一般称为“网卡”。

在计算机内部，网卡与CPU之间的通信一般是通过计算机主板上的I/O总线以并行传输方式进行。网卡与外部以太网(局域网)之间的通信，一般是通过传输媒体(同轴电缆、双绞线电缆、光纤)以串行方式进行。

网卡除了要实现物理层的数据链路层功能，另外一个重要功能是进行串行传输和并行传输的转换，由于网络的传输速率和计算机内部总线上的传输速率并不相同，因此在网卡的核心芯片中都会包含用于缓存数据的存储器。

在确保网卡硬件正确的情况下，为了使网卡正常工作，还必须要在计算机的操作系统中为网卡安装相应的设备驱动程序，驱动程序负责驱动网卡发送和接收帧。

```
              计算机                                             计算机
             并行传输        串行传输           串行传输        并行传输
 CPU和存储器<--------->网卡<---------->局域网<----------->网卡<-------->CPU和存储器
```



##### 网络适配器构成

```
核心芯片：实现以太网的数据链路层和物理层功能
EEPROM：存储MAC地址和网卡的相关信息的电可擦可编程只读存储器
BootROM插槽：安装用于网络无盘工作站启动的BootROM芯片
网络隔离变压器：隔离核心芯片和RJ45网络接口，以便提高抗干扰能力并对核心芯片进行防雷击保护
RJ45网络接口：连接双绞线电缆
PCI接口：连接计算机主板
```



##### 网络适配器工作模式

广播模式：它的物理地址（MAC）地址是 0Xffffff 的帧为广播帧，工作在广播模式的网卡接收广播帧。

多播模式：多播传送地址作为目的物理地址的帧可以被组内的其它主机同时接收，而组外主机却接收不到。但是，如果将网卡设置为多播传送模式，它可以接收所有的多播传送帧，而不论它是不是组内成员。

直接模式：只接收目的地址是自己Mac地址的帧。

混杂模式：能够接收一切通过它的数据，而不管该数据目的地址是否是它，这实际上就是Sniffer工作的基本原理：让网卡接收一切他所能接收的数据。当网卡处于这种"混杂"方式时，该网卡具备"广播地址"，它对所有遇到的每一个数据帧都 产生一个硬件中断以便提醒操作系统处理流经该物理媒体上的每一个报文包。可见，Sniffer工作在网络环境中的底层，它会拦截所有的正在网络上传送的数 据，并且通过相应的软件处理，可以实时分析这些数据的内容，进而分析所处的网络状态和整体布局。

ifconfig eth0 promisc 设置混杂
ifconfig eth0 -promisc　取消混杂



#### MAC地址

当多个主机连接在同一个广播信道上，要想实现两个主机之间的通信，则每个主机都必须有一个唯一的标识，即一个数据链路层地址。

在每一个主机发送的帧的首部中，都携带有发送主机(源主机)和接收主机(目的主机)的数据链路层地址。由于这类地址是用于媒体接入控制(Medium Access Control,MAC)的，因此被称为MAC地址。

MAC地址一般被固化在网卡的电可擦可编程只读存储器EEPROM中，因此MAC地址也被称为硬件地址，也被称为物理地址。

一般情况下普通用户计算机中包含两块网卡：用于接入有线局域网的以太网卡和用于接入无线局域网的WIFI网卡。每一块网卡都有一个全球唯一的MAC地址。

综上所述，MAC地址是对网络上各接口的唯一标识，而不是对网络上各设备的唯一标识。



##### IEEE 802局域网MAC地址格式

从左至右一共六个字节

```
第一字节                            第二到五字节                     第六字节
b7b6b5b4b3b2b1b0                   ...........                   b7b6b5b4b3b2b1b0
##第一到三字节是组织唯一标识符OUI，生产网络设备的厂商向IEEE注册管理机构申请一个或多个OUI。
##第四到六字节是网络接口标识符，厂商可自行随意分配的网络接口标识符，只要保证生产出的网络设备没有重复地址即可。
将每四个比特写成一个十六进制的字符，共十二个字符，将每两个字符分为一组，共六组，各组之间用短线连接，得到MAC地址标准表示法：XX-XX-XX-XX-XX-XX(Windows)
其他表示法：把短线改成冒号XX:XX:XX:XX:XX:XX（Linux，Android，IOS）；将每四个字符分为一组，共三组，各组之间用"."进行连接XXXX.XXXX.XXXX(Packet Tracer仿真软件)
```

MAC地址中组织唯一标识符OUI也称为公司标识符，可在IEEE官网查询到公司标识符的分配情况。

IEEE规定：MAC地址的第一字节的b0(最后一位)为I/G(Individual/Group)位，I/G位为0时，表示MAC地址是单播地址；I/G位为1时，表示MAC地址是多播地址。

考虑到有厂商未向IEEE注册管理机构购买组织唯一标识符OUI，IEEE规定MAC地址第一字节的b1位(倒数第二位)为G/L(Global/Local)位，G/L位为0时，MAC地址是全球管理的，确保在全球没有相同的MAC地址；G/L位为1时，MAC地址是本地管理的，用户可任意分配MAC地址

##### IEEE 802局域网的MAC地址发送顺序

字节发送顺序：第1字节到第六字节

##### 使用案例

单播MAC地址

源主机想要向目的主机发送单播帧，需要在帧首部中目的地址字段填入对应主机的MAC地址，源地址字段填入自己的MAC地址。源主机把该单播帧发送到总线型以太网中，该条总线上所有主机都会收到单播帧。非目的主机在收到时发现目的MAC地址与自己的MAC地址不匹配，就会丢弃该帧。目的主机在收到时发现目的MAC地址与自己的MAC地址匹配，就会接受该帧，并将该帧的数据载荷交付上层处理。



广播MAC地址

源主机想要发送广播帧，需要在帧首部中目的地址字段填入广播MAC地址FF-FF-FF-FF-FF-FF，源地址字段填入自己的MAC地址。源主机把该广播帧发送到总线型以太网中，该条总线上所有主机都会收到广播帧。所有主机的网卡发现该帧的目的MAC地址是广播MAC地址，就会接受该广播帧，并将该帧的数据载荷交付上层处理。



多播MAC地址

为提高网络的可靠性，多个交换机连接形成一个环路。然而环路会造成广播帧在环路中永久兜圈的问题。为解决该问题，交换机之间会交互一种特殊的帧，即BPDU（网桥协议数据单元）。各交换机根据BPDU中的一些参数按照生成树协议所使用的生成树算法，最终某些交换机会阻塞自己的某些接口，以此来破除环路。BPDU首部中的目的地址就是生成树协议专用的多播MAC地址。



#### 网卡对MAC地址的处理

网卡从网络中每收到一个无误码的帧，就检查帧首部中的目的MAC地址，按以下情况处理：

(1)如果目的MAC地址是广播地址，就接受该帧

(2)如果目的MAC地址与网卡上固化的全球单播MAC地址相同，就接受该帧。

(3)如果目的MAC地址是网卡支持的多播地址，就接受该帧

(4)除上述(1)、(2)、(3)情况外，丢弃该帧。



网卡还可以被设置为一种特殊的工作方式：混杂模式(Promiscuous Mode)。工作在混杂模式的网卡只要收到共享媒体上传来的帧就会收下，而不管帧的目的MAC地址是什么。嗅探器(Sniffrt)就是一种工作在混杂模式的网卡，再配合相应的工具软件，比如WireShark，就可以作为一种非常有用的网络工具来学习和分析网络。



**注：全球单播MAC地址就如同身份证上的身份证号码，具有唯一性，它往往与用户个人信息绑定在一起。因此，用户尽量确保自己拥有的全球单播MAC地址不被泄露。为避免用户设备连接WiFi热点时MAC地址泄露的安全问题，目前大多数移动设备都已经采用了随机MAC地址技术。**



### 8.CSMA/CD

为解决各站点争用总线的问题，共享总线以太网使用了一种专用协议——CSMA/CD(Carrier Sense Mutiple Access with Collision Detection，载波监听多址接入/碰撞检测)



#### 基本原理

多址接入：多个站点连接在一条总线上，竞争使用总线

载波监听：每个站点在发送帧之前，先要检测一下总线上是否有其他站点在发送帧(“先听后说”)：若检测到总线空闲96比特时间(发送96比特所耗费的时间，也称为帧间最小间隔)，则发送这个帧；若检测到总线忙，则继续检测并等待总线转为空闲96比特时间，然后发送这个帧。

碰撞检测：每个正在发送帧的站点边发送边检测碰撞("边说边听")：一旦发现总线上出现碰撞，立即停止发送，退避一段随机时间后再次从载波监听开始进行发送

先听后发，边听边发，冲突停发，随机重发



##### 多址接入

多个站点连接到一根总线上，各主机随机发送帧

##### 载波监听

其中一个站点要发送帧，先进行载波监听，在检测到总线空闲96比特时间后，就可以发送帧了。帧的信号会沿着总线进行传播。当一个站点使用总线发送帧的过程中，有一个站点也要发送帧。这个站点进行载波监听，发现总线忙，于是持续监测总线，一旦发现总线空闲96比特时间，则立即发送帧。

##### 碰撞检测

在发送帧的同时，进行检测碰撞。只要没检测到碰撞，就一直发送帧的剩余部分。

若站点在发送帧的过程中，有另一个站点也要发送帧，后一站点进行载波监听，检测到总线空闲，但总线实际上并不空闲（前一站点发送的信号未传播到后一站点，后面站点未检测到），后一站点检测到总线空闲96比特时间后，就立即发送帧，这必然会产生碰撞。在产生碰撞的时刻，这两个站点都在边发送帧边检测碰撞，但都检测不到碰撞。碰撞信号沿总线传播，其中一个站点较早的检测到碰撞并停止发送帧，退避一段随机时间后，重新从载波监听开始，以便重发之前遭遇到碰撞的帧；当另一站点检测到碰撞后，立即停止发送，退避一段随机时间后，重新从载波监听开始，以便重发之前遭遇到碰撞的帧。

以太网还采取了一种叫强化碰撞的措施：发送帧的站点一旦检测到碰撞，除了立即停止发送帧外，还要再继续发送32比特或48比特的认为干扰信号，以便有足够多的碰撞信号使所有站点都能检测出碰撞。



#### 说明

```
载波监听检测到总线空闲，但总线并不一定空闲。

使用CSMA/CD协议的共享总线以太网的各站点，只是尽量避免碰撞并在出现时做出退避后重发的处理，但不能完全避免碰撞。

在使用CMDA/CD协议时，由于正在发送帧的站点必须“边发送帧边检测碰撞”，因此站点不可能同时进行发送和接收，也就是不可能进行全双工通信，而只能进行半双工通信(双向交替通信)
```



### 9.使用集线器的共享式以太网

集线器(Hub)的特点：

```
使用集线器的以太网虽然物理拓扑是星型的，但是在逻辑上仍然是一个总线网，总线上各站点共享总线资源，还是在使用CSMA/CD协议。
集线器只工作在物理层，它的每一个接口仅简单地转发比特，并不进行碰撞检测，碰撞检测的任务由各站点的网卡负责。
集线器一般都有少量的容错能力和网络管理功能，例如，若网络中某个站点的网卡出现了故障而不停的发送帧，集线器可以检测到这个问题，在内部断开与出故障网卡的连接，使整个以太网能正常工作。
```



与使用细同轴电缆的共享总线型以太网的异同

物理拓扑不相同，但是工作逻辑相同，网络中各站点都使用CSMA/CD协议来共享(争用)网络资源。

以网络中某一主机向另一主机发送帧为例：

使用细同轴电缆的共享总线型以太网：表示帧的信号会沿着总线传播到各个主机。

使用集线器和双绞线的星型拓扑的共享式以太网：集线器收到帧后，会把帧从自己的其他接口转发出去，这样表示帧的信号也会传播到各个主机。

若网络中有多个主机要发送帧，这两种以太网都会出现碰撞的情况。



IEEE制定了10BASE-T星型以太网的标准802.3i，使以太网在局域网中的统治地位奠定了牢固的基础。10指传输速率为10Mbps，BASE是指采用基带信号进行传输，T是指采用双绞线作为传输媒体。10BASE-T以太网的通信距离较短，每个站点到采集器的距离不能超过100m。802.3以太网还可以使用光纤作为传输媒体，相应的标准是10BASE-F，F是指光纤，光纤主要用作集线器之间的远程连接。



### 10.交换式以太网

1990年面世的交换式集线器(Switching Hub)，实质上是具有多个接口的网桥，常称为以太网交换机(Switch)或二层交换机。“二层”是指以太网交换机工作在数据链路层(包括物理层)，与网桥相同，交换机内部的转发表也是通过自学习算法，基于网络中各主机之间的通信，自动地逐步建立起来的。同时，交换机也使用生成树协议STP，来产生能够连通全网但不产生环路的通信路径。

仅使用交换机而不是用集线器的以太网就是交换式以太网。处于同一个冲突域的为共享式以太网，分割了两个冲突域的为交换式以太网。



#### 以太网交换机

以太网交换机本质上就是一个多接口的网桥，交换机自学习和转发帧的流程与网桥是相同的，另外，交换机也使用生成树STP协议来产生能够连通全网但不产生环路的通信路径。以太网交换机收到帧以后，在转发表中查找帧的目的MAC地址所对应的接口号，然后通过该接口转发帧，所以转发决策使用的PDU地址是目的物理地址。

当交换机的接口与计算机或交换机连接时，可以工作在全双工模式下，并能在自身内部同时连通多对接口，使每一对相互通信的计算机都能像独占传输媒体那样，无碰撞的传输数据，这样就不需要使用CSMA/CD协议了。

当交换机的接口连接的是集线器时，该接口就只能使用CSMA/CD协议并只能工作在半双工模式下。

现在的交换机和计算机中的网卡都能自动识别上述两种情况，并自动切换到相应的工作方式。

交换机一般具有多种速率的接口，例如10Mbps，100Mbps，1Gbps甚至10Gbps的接口，大部分接口支持多速率自适应。



一般的交换机都采用"存储转发"方式，为了减少交换机的转发时延，某些交换机采用了直通(Cut-Through)交换方式。采用直通交换方式的交换机，在接收帧的同时就立即按帧的目的MAC地址决定该帧的转发接口，然后通过其内部基于硬件的交叉矩阵进行转发，而不必把整个帧先缓存后在进行处理(只要接收完以太网帧的目的MAC地址就可以将帧直接转发到目的端口)。直通交换的时延非常小，直通交换不检查差错就直接将帧转发出去，以后可能会将一些无效帧转发给其他主机。



#### 共享式以太网和交换式以太网的对比

共享式以太网和交换式以太网的对比主要是对比集线器和交换机

```
假设交换机的转发表已学习到了所有主机与自己各接口的对应关系

主机A向主机B发送单播帧
共享式：单播帧会通过集线器传播到网络中的其他各主机，其他各主机中的网卡会根据单播帧的目的MAC地址，来决定接受或丢弃该帧。
交换式：交换机收到单播帧后，根据帧的目的MAC地址和自身的转发表，将帧明确地转发给目的主机，而不是网络中的其他各主机。目的主机中的网卡根据单播帧的目的MAC地址决定接受该帧。

主机A发送广播帧：
共享式：广播帧会通过集线器传播到网络中的其他各主机，其他各主机中的网卡检测到帧地目的MAC地址是广播地址，就接受该帧。
交换式：交换机收到广播帧后，检测到帧的目的MAC地址是广播地址，于是从除该帧进入交换机地接口其他所有接口转发该帧，网络中其他各主机中的网卡收到该帧后，检测到帧地目的MAC地址是广播地址，就接受该帧。
可以看出共享式以太网中的各主机属于同一个广播域，而交换式以太网中的各主机也属于用一个广播域。但它们地原理不同：集线器只工作在物理层，它并不能识别帧首部中的目的MAC地址，不管集线器收到的单播帧还是广播帧，集线器仅仅是将表示帧的信号从其他所有接口转发出去；交换机工作在数据链路层，可以识别帧的结构，从帧首部中提取出目的MAC地址，然后查找自己的转发表，根据查表结果对帧进行明确转发或盲目转发或丢弃。

多对主机之间同时通信：
主机A给主机B发送单播帧地同时主机C给主机D发送单播帧
共享式：当多对主机同时通信时，必然会产生碰撞，遭遇碰撞的帧会传播到网络中的各主机，各主机中的网卡检测到遭遇碰撞的帧后，将其丢弃。
交换式：由于交换机对收到的帧进行存储转发，并且能实现多接口的高速并行交换，因此不会产生碰撞。A发送给B的单播帧以及C发送给D的单播帧都会被交换机正确转发，主机B和D中的网卡接受各自收到的单播帧。
```



两个独立的共享式以太网，各自都是一个独立的广播域和碰撞域。若用集线器将这两个独立的共享式以太网连接起来，则会形成一个更大的广播域和碰撞域的共享式以太网，即既扩大了广播域也扩大了碰撞域；若用交换机将这两个独立的共享式以太网连接起来，则会形成一个具有更大广播域但原本独立了的两个碰撞域仍被交换机隔离的以太网，即扩大了广播域，隔离了碰撞域



### 11.以太网的MAC帧格式

以太网MAC帧格式分为DIX Ethernet V2的帧格式和IEEE 802.3 的帧格式，这两种格式差别很小，仅"类型"字段有差别

```
                              以太网V2的MAC帧
目的地址      源地址         类型           数据载荷                FCS
  6B           6B            2B           46B~1500B               4B
目的地址和源地址字段分别为6字节长，用来填充帧的目的MAC地址和源MAC地址
类型字段为2字节长，其值用来指明数据载荷中的内容是由上一层的哪一个协议封装的，以便将收到的MAC帧的数据载荷上交给上一层的这个协议。
帧检验序列FCS字段4字节长，他的内容是使用CRC生成的帧检验序列，接收方的网卡通过该字段的内容就可检测出帧在传输过程中是否产生了误码，范围是从目的地址开头到数据载荷结尾
```

数据链路层将封装好的MAC帧交付给物理层进行发送，物理层在发送MAC帧之前，在其前面添加8字节的前导码。前导码中前7个字节为前同步码， 作用是使接收方的时钟同步，之后的1个字节为帧开始定界符，表明其后面紧跟着的就是以太网帧。

接收方可能收到的无效MAC帧包括：MAC帧的长度不是整数个字节；通过MAC帧的FCS字段的值检测出有误码；MAC帧的长度不再64~1518字节之间。接收方收到无效的MAC帧时，就简单将其丢弃，以太网的数据链路层没有重传机制。



### 12.虚拟局域网VLAN

虚拟局域网(Virtual Local Area Network，VLAN)是一种将局域网内的站点划分成与物理地址无关的逻辑组的技术，一个逻辑组就是一个VLAN，VLAN中的各站点具有某些共同的应用需求。

属于同一VLAN的站点之间可以直接进行通信，而不同VLAN中的站点之间不能直接通信。网络管理员可对局域网中的各交换机进行配置来建立多个逻辑上独立的VLAN。连接在同一交换机上的多个站点可以属于不同的VLAN，而属于同一VLAN的多个站点可以连接在不同的交换机上。虚拟局域网VLAN并不是一种新型网络，他只是局域网能够提供用户的一种服务。



#### 虚拟局域网VLAN的实现机制

虚拟局域网VLAN有多种实现技术，最常见的就是基于以太网交换机的接口来实现VLAN，这就需要以太网交换机能够实现以下功能：能够处理带有VLAN标记的帧，也就是IEEE 802.1Q帧；交换机的各接口可以支持不同的接口类型，不同接口类型的接口对帧的处理方式有所不同。



##### IEEE 802.1Q帧

IEEE 802.1Q帧也称为Dot One Q帧，它对以太网V2的MAC帧格式进行了扩展：在源地址字段和类型字段之间插入了4字节的VLAN标签(tag)字段。

```
目的地址        源地址           VLAN标签        类型          数据载荷          FCS
6Byte          6Byte            4Byte          2Byte         46~1500Byte      4Byte

VLAN组成：
TPID        PRI          CFI           VID
16Byte      3Byte        1Byte         12Byte
标签协议标识符TPID：长度16比特，其值固定为0x8100，表示该帧是IEEE 802.1Q帧
优先级PRI：长度为3比特，取值范围是0~7，值越大优先级越高，当网络阻塞时，设备优先发送优先级高的IEEE 802.1Q帧。
规范格式指示符CFI：长度为1比特，取值为0表示MAC地址以规范格式封装，取值为1表示MAC地址以非规范格式封装，对于以太网，CFI取值为0。
虚拟局域网标识符VID：长度为12比特，取值范围是0~4095，其中0和4095保留不使用。VID是802.1Q帧所属VLAN的编号，设备利用VID来识别帧所属的VLAN，广播帧只在同一个VLAN内转发，这样就将广播帧限制在了一个VLAN内。
```



802.1Q帧一般不由用户主机处理，而是由以太网交换机来处理：

```
当交换机收到普通的以太网MAC帧时，会给其插入4字节的VLAN标签使之成为802.1Q帧，该处理简称为“打标签”
当交换机转发802.1Q帧时，可能会删除其4字节的VLAN标签使之成为普通的以太网MAC帧，该处理简称为“去标签”。交换机转发802.1Q帧时也有可能不进行“去标签”处理，是否进行“去标签”处理取决于交换机的接口类型。
```



##### 以太网交换机的接口类型

根据接口在接收帧和发送帧时对帧的处理方式的不同，以及接口连接对象的不同，以太网交换机的接口类型一般分为Access和Trunk两种。

当以太网交换机上电启动后，若之前未对其各接口进行过VLAN的相关设置，则各接口的接口类型默认为Access，并且各接口的缺省值VLAN ID为1，即各接口默认属于VLAN1。对于思科交换机，接口的缺省值VLAN ID成为本征VLAN(Native VLAN)；对于华为交换机，接口的缺省值VLAN ID成为端口VLAN ID(Port VLAN ID)，简记为PVID。交换机的每个接口有且仅有一个PVID。

```
Access接口：
Access接口一般用于连接用户计算机，由于其只能属于一个VLAN，因此Access接口的PVID值与其所属VLAN的ID相同，其默认值为1。
接受处理：
一般只接受“未打标签”的普通以太网MAC帧，根据接收帧的接口的PVID给帧“打标签”。即插入4字节的VLAN标签字段，VLAN标签字段中的VID取值就是接口的VID值。
转发处理：
若帧中的VID值与接口的PVID值相等，则给帧“去标签”后再进行转发，否则不转发帧。因此，从Access接口转发出的帧，是不带VLAN标签的普通以太网MAC帧。

Trunk接口：
Trunk接口一般用于交换机之间的互连。Trunk接口可以属于多个VLAN，即Trunk接口可以通过属于不同VLAN的帧，Trunk接口的默认PVID值为1，一般不建议用户修改，若互连的Trunk接口的PVID值不相同，则可能出现转发错误。
接受处理：
既可以接受“未打标签”的普通以太网MAC帧，也可以接收“已打标签”的802.1Q帧。若接收到普通以太网MAC帧时，根据接收到的接口PVID给帧“打标签”。这与Access接口处理相同。
转发处理：
对于帧的VID值扽与接口的PVID的值802.1Q帧。将其“去标签”转发；对于帧的VID值不等于接口的PVID值的802.1Q帧，将其直接转发，因此，从Trunk接口转发出的帧，可能是普通以太网MAC帧，也可能是802.1Q帧。
```



## 三、网络层

### 1.网络层中重要概念

网络层的主要任务就是将分组从源主机经过多个网络和多段链路传输到目的主机，可以将该任务划分为分组转发和路由选择两种重要功能。分组转发属于数据层面，每一个路由器根据本路由器生成的转发表，把收到的分组从查找到的对应接口转发出去。路由选择属于控制层面，路由器必须和相邻的路由器根据路由选择协议所用的路由算法计算路由，经常交换路由信息，才能创建出本路由器的路由表。

网络层向其上层提供的两种服务：面向连接的虚电路服务和无连接数据包服务。

面向连接的虚电路服务核心思想是“可靠通信应由网络自身来保证”，两台主机在通信时必须先建立网络层连接——虚电路，以保证通信双方所需的一切网络资源。后面通信双方沿着已建立的虚电路发送分组。虚电路表示这是一条逻辑上的连接，分组沿着这条逻辑连接按照存储转发方式传输，而不是真正建立了一条物理连接。而采用电路交换的电话通信，则是先建立一条真正的物理连接，因此，分组交换的虚拟连接与电路交换的连接只是类似，但并不完全一样。分组的首部仅在连接建立阶段使用完整的目的主机地址，之后每个分组的首部只需要携带一条虚电路编号即可。通信结束后，通信双方需要释放之前所建立的虚电路。这种通信方式如果在使用可靠传输的网络协议，就可以使所发送的分组最终正确(无差错按顺序到达，不丢失，不重复)到对接收方。

无连接的数据包服务核心思想是“可靠通信应由用户主机来保证”，当两台主机通信时，不需要建立网络层连接，每个分组可以走不同的路径。因此，每个分组的首部都必须携带目的主机的完整地址。通信结束后，通信双方没有需要释放的连接。这种方式将相对简单的尽最大努力(即不可靠)的分组交付功能置于因特网核心。将复杂的网络处理能力置于因特网的边缘(即用户主机和其内部的运输层)。由于网络自身不提供端到端的可靠传输服务，这就使得网络中的路由器可以做的比较简单，大大降低了网络造价。

分组从源节点发送到目的节点，若中间必须经过一个或几个路由器，是间接交付；不需要经过路由器，是直接交付。

分组在传送途中每一次转发被称为一**"跳"**(hop)，路由器在转发分组时也常常使用**“下一跳”**的说法。

### 3.异构网络互连

如果全世界范围内把数以百万计的网络都互连起来，并能狗相互通信，这样的任务是比较复杂的，因为没有一种单一的网络能够适应所有用户的需求。将网络互相连接起来需要使用一些中间设备，物理层使用转发器，数据链路层使用网桥或桥接器以及交换机，网络层使用路由器，网络层以上使用网关。由于所有参加互连的计算机网络都是用网际协议IP，因此可以把互连以后的计算机网络看成一个虚拟互联网络。



### 4.IPv4

IPv4地址是给因特网上每一个主机(或路由器)的每一个接口分配的一个在全世界范围内唯一的32比特的标识符。IPv4地址由因特网名字和数字分配机构(Internet Corporation Assigned Names and Numbers，ICANN)进行分配。



#### IPv4的表示方法

 IPv4地址采用点分十进制展示代替32比特构成的初始表示，方便用户使用。8位无符号数二进制整数转换成十进制数时，计算每一位的权值然后再相加。在十进制正整数转8位无符号二进制数时，通过模二取余然后逆序输出，例如：130二进制表示方法是130/2=65，余0；65/2=32，余1；32/2=16，余0；16/2=8，余0；8/2=4，余0；4/2=2，余0；2/2=1，余0；1/2=0，余1。最终十进制数130用二进制表示是：10000010。



IPv4的三种编址方式：分类编址、划分子网和无分类编址

#### 分类编址

32比特的IPv4的地址分为网络号和主机号。网络号标志主机(或路由器)的接口所连接到的网络，同一个网络中，不同主机(或路由器)的接口的IPv4地址的网络号必须相同，表示它们属于同一个网路。主机号标志主机(或路由器)的接口，同一个网络中，不同主机(或路由器)的接口的IPv4地址的网络号必须各不相同，以便区分各主机(或路由器)的接口。

#### 划分子网

IPv4分类编址方法不灵活、容易造成大量IPv4地址资源浪费。

划分子网的编址方法结构：网络号，子网号，主机号

子网掩码可以表明分类IPv4地址的主机号部分被借用了几个比特作为子网号。与IPv4类似，子网掩码也是由32比特构成的，左起多个连续的比特1对应IPv4地址中的网络号和子网号，后面多个连续的比特0对应IPv4地址中的主机号。将划分子网的IPv4地址与对应的子网掩码进行逐比特的逻辑与运算，就可以得到该IPv4地址所在子网的网络地址。原理：IPv4的网络号和子网号保持不变，而主机号全部清零，就变成了网络地址。



##### 默认子网掩码

默认子网掩码是指未划分子网的情况下使用的子网掩码。

```
A类地址的默认子网掩码是255.0.0.0
B类地址的默认子网掩码是255.255.0.0
C类地址的默认子网掩码是255.255.255.0
```



#### 无分类编址

无分类编址的全称是无分类域间路由选择(Classless Inter-Domain Routing，CIDR)，CIDR消除了传统A类，B类和C类以及划分子网的概念。CIDR可以更加有效地分配IPv4地址资源，并且可以在IPv6使用前允许因特网的规模继续增长。

无分类域间路由选择结构：网络前缀，主机号。网络前缀是不定长的，仅从IPv4地址自身是无法确定其网络前缀和主机号的，需要配合使用32比特的地址掩码来指明长度。

无分类编址方法使用的地址掩码与划分子网使用的子网掩码类似，由32比特构成。左起多个连续的比特1对应IPv4地址中的网络前缀，后面多个连续的比特0对应IPv4地址中的主机号。

CIDR使用“斜线记法”，即在IPv4地址后面加上斜线，在斜线之后写上网络前缀所占的比特数量。无分类域间路由选择CIDR是将网络前缀都相同的、连续的多个无分类IPv4地址，组成一个CIDR地址块，只要知道CIDR地址块中的任何一个地址，就可以知道该地址块以下信息：地址块中最小地址，地址块中最大地址，地址块中地址数量，地址块中聚合某类网络(A类，B类，C类)的数量，地址掩码。

使用CIDR可以根据用户的需求来分配适当大小的CIDR地址块，因此可以更加有效地分配IPv4地址空间。



##### 构造超网

使用CIDR也可以实现路由聚合(也称为构造超网)。

一个大的CIDR地址块中往往包含很多小地址块，所以在路由器的转发表中就利用较大的一个CIDR地址块来代替许多较小的地址块，这使得转发表中只有一个项目就可以表示原来传统分类地址的很多个路由项目，大大压缩了转发表所占的空间，减少了查找转发表所需时间。

网络前缀越长，地址块越小，路由越具体。若路由器查表转发分组时发现有多条路由条目匹配，则选择网络前缀最长的那条路由条目，这称为最长前缀匹配，因为这样的路由更具体。



#### IPv4地址的应用规划

IPv4地址的应用规划是指将给定的IPv4地址块(或分类网络)划分成若干个更小的地址块(或分类网络)，并将这些地址块(或子网)分配给互联网中不同网络，进而可以给各网络中的主机和路由器的接口分配IPv4地址。

一般有两种方法

```
定长的子网掩码(Fixed Length Subnet Mask，FLSM)
所划分出的每一个子网都是用同一个子网掩码。每个子网所分配的IP地址数量相同，但是容易造成地址资源的浪费。

变长的子网掩码(Variable Length Subnet Mask，VLSM)
所划分出的每一个子网可以使用不同的子网掩码，每个子网所分配的IP地址数量可以不同，这样尽可能地减少了对地址资源的浪费，在地址块中选取子块原则：每个子块地起点位置不能随便选取，只能选取主机号部分是块大小整数倍地地址作为起点，最好先为大的子块选取。
```



#### IPv4地址与MAC地址

##### IPv4地址与MAC地址的封装位置

MAC地址是固化在网卡上的的ROM中，是物理地址；对应的逻辑地址是IP地址。IP地址属于网际层，在网际层的IP数据报报文的IP首部中封装源IP地址和目的IP地址；MAC地址属于数据链路层，在数据链路层的MAC帧帧首部封装源MAC地址和目的MAC地址。在网络层和网络层以上使用的是IP地址，数据链路层及以下使用的是MAC地址。



##### 数据包传送过程中IPv4与MAC地址的变化情况

关注的重点是网际层在封装IP数据报时源IP地址和目的IP地址应该填写什么；数据链路层在封装帧时源MAC地址和目的MAC地址应该填写什么

```
主机H1            路由器R1            路由器R2            主机H2
IP1            IP3      IP4        IP5      IP6          IP2
MAC1          MAC3      MAC4       MAC5     MAC6         MAC2

IP数据报：
     源：IP1                 源：IP1             源：IP1
     目：IP2                 目：IP2             目：IP2

MAC帧
     源：MAC1                源：MAC4            源：MAC6
     目：MAC3                目：MAC5            目：MAC2
     
可以看出：在数据包的传送过程中，数据报的源IP地址和目的IP地址保持不变；源MAC地址和目的MAC地址逐个链路(或逐个网络)改变
```



##### IPv4地址与MAC地址的关系

如果仅使用MAC地址进行通信，会出现以下问题：
	因特网中的每台路由器的路由表中就必须记录因特网上所有主机和路由器各接口的MAC地址。
	手动给各路由器配置路由表几乎是不可能完成的任务，即使使用路由协议让路由器通过相互交换路由信息来自动构建路由表，也会因为路由信息需要包含海量的MAC地址信息而严重占用通信资源。
	包含海量MAC地址的路由信息需要路由器具备极大的存储空间，而且会给分组的查表转发带来非常大的时延。

因特网的网际层使用IP地址进行寻址，就可使因特网中个路由器的路由表中的路由记录的数量大大减少，因为只需记录部分网络的网络地址，而不是记录每个网络中各通信设备的各接口的MAC地址。路由器收到IP数据报后，根据其首部中的目的IP地址的网络号部分，基于自己的路由表进行查表转发。

查表的结果可以指明IP数据报的下一跳路由器的IP地址，但无法指明该IP地址所对应的MAC地址。因此，在数据链路层封装该IP地址数据报成为帧时，帧首部中的目的MAC地址字段就无法填写，这个问题需要使用网际层中的地址解析协议ARP来解决。



### 5.地址解析协议ARP

地址解析协议ARP是用于已知一个机器的IP地址，找到对应的MAC地址。

具体方法：主机的ARP高速缓存中存放一个从IP地址到MAC地址的映射表，并且这个映射表还经常新增或者超市删除来动态更新。(更新原因：ARP高速缓存表中的IP地址与MAC地址的对应关系并不是永久不变的，因此通过ARP协议获取到的IP地址与MAC地址的对应关系记录有生命周期)

**不能跨网段使用ARP，而是要逐段链路或者逐个网络使用ARP**

查找对应主机IP地址和MAC地址映射关系过程

```
                     主机B
主机A                交换机                主机C

背景：A想给B发送一个分组。A知道B的IP地址，但是不知道B的MAC地址，高速缓存表中没有B的信息

1.A在当前局域网广播一个ARP请求分组来获取B的MAC地址，报文内容”我的IP地址是xxx.xxx.xxx.xxx，MAC地址是xx-xx-xx-xx-xx-xx，我想知道IP地址是yyy.yyy.yyy.yyy对应的MAC地址“
2.交换机收到封装有该ARP请求报文的以太网广播帧以后，将其从自己的其他所有接口转发出去
3.在该局域网内的所有主机网卡收到该广播帧，将其交付上层进行处理。主机C网卡接收到该广播帧后，上层的ARP进程解析该ARP请求报文，发现所询问的IP地址不是自己的地址，因此不予理会；主机B网卡接收到该广播帧后，上层的ARP进程解析该ARP请求报文，发现所询问的IP地址正是自己的IP地址，需要进行响应。
4.主机B先将ARP请求报文中所携带的主机A的IP地址和MAC地址记录到自己的ARP高速缓存表中，然后给主机A发送ARP响应报文，来告知主机A自己的MAC地址。
5.主机B给主机A发送单播类型ARP响应报文，封装在以太网帧中发送，帧的目的MAC地址为主机A的MAC地址，ARP响应报文内容是“我的IP地址是yyy.yyy.yyy.yyy，我的MAC地址是yy-yy-yy-yy-yy-yy”。
6.交换机收到封装有该ARP响应报文的以太网单播帧后，将其明确转发给主机A。主机A的网卡收到该单播帧后，发现其目的MAC地址就是自己的MAC地址，于是接受该单播帧，上层解析该ARP请求报文，将其所包含的主机B的IP地址与MAC地址记录到自己的ARP高速缓存表中。
7.主机A可以给主机B发送分组。
```



高速缓存表中每一条都有其类型

```
动态：通过ARP自动获取，生命周期默认为两分钟，当生命周期结束时，该记录将自动删除。
静态：由用户或网络管理员手动配置，不同操作系统下的生命周期不同，例如系统重启后不存在或系统重启后仍然有效
```



注意事项：
ARP协议可以划归在网际层，也可以划归在数据链路层。
除了ARP请求报文和响应报文，ARP还有其他类型报文，例如用于检查IP地址冲突的“无故ARP”
ARP协议没有安全验证机制，存在ARP欺骗和攻击等问题。



### 6.IP数据报的发送和转发流程

IP数据报的发送和转发过程包含以下两个过程：

主机发送IP数据报和路由器转发IP数据报



```
         交换式以太网1                    交换式以太网2
             A                                 D
B            S1               R                S2            E
             C                                 F
    地址块192.168.0.0/25              地址块192.168.0.128/25

源主机如何判断出目的主机是否和自己在同一个网络中？
假设C要给F发送IP数据报，C知道F的IP地址，于是将F的IP地址的前25各比特取出，与C自己的IP地址的前25个比特的网络前缀进行比较(要发送的分组的目的地址和本网络的子网掩码按位进行AND运算)，可以发现双方前25个比特的网络前缀不完全相同，这表明C和F不在同一个网络中。

源主机如何知道需要将IP数据报交给哪个路由器来转发？
用户为了让本网络中的各主机能与其他网络中的各主机进行通信，就必须给本网络中的各主机指定本网络中的一个路由器，由该路由器帮忙进行转发，所指定的路由器也被称为默认网关。当本网络中的主机，要和其他网络中的主机进行通信时，会将IP数据报发送给默认网关，由默认网关帮主机把IP数据报转发出去。

路由器收到某个正确的IP数据报(首部各字段合法且传输过程中无误码)，应该如何转发？
路由器基于IP数据报首部中的目的IP地址在路由表中进行查询：
	若找到匹配的路由条目，则按照该路由条目的指示进行转发
	否则丢弃该IP数据报，并向发送该IP数据报的源主机发送ICMP差错报告。
路由器根据IP数据报的目的IP地址，在自己的路由表中查找匹配的路由条目，将IP数据报的目的地址与路由条目中的地址掩码进行逐比特逻辑与运算，直到找匹配的路由条目，路由器根据匹配的路由条目中的下一跳的指示，从路由器的其他接口直接交付给目的主机。
路由器只需通过ARP协议获取到与自己的非传入的接口处于同一网络中的目的主机的MAC地址，就可将IP数据报封装成帧后发送给目的主机。

路由器对广播IP数据报的处理
一个主机发送了一个广播IP数据报，该IP数据报首部中目的地址字段的值，设置为本网络的广播地址，或者设置为受限的广播地址255.255.255.255。该主机所在网络中的各主机和连接该以太网路由器的接口，都会收到该广播IP数据报。默认情况下，路由器不转发广播IP数据报，也就是说路由器隔离广播域。如果因特网中数量巨大的路由器收到广播IP数据报后都进行转发，则会造成巨大的广播风暴，严重浪费因特网资源。
同时，在用户私有网络中，用户可以根据应用需求对路由器进行设置，shi'qi

### 7.IPv4数据报的首部格式

IPv4数据报的首部格式及其内容是实现IPv4协议各种功能的基础。在TCP/IP标准中，各种数据格式常常以32比特(即4字节)为单位来描述。

IPv4数据报首部格式

```
每一格称为字段或者域
b 0          4           8                   16     19                                    31
  版本       首部长度     区分服务             总长度
  标识                                        标志   片偏移
  生存时间                协议                 首部检验和
  源IP地址
  目的IP地址
  可选字段(长度可变)                                                        填充

从版本到目的IP地址为固定部分，共20字节；可选字段到填充部分为可变部分，最大40字节。
标识、标志和片偏移共同用于IPv4数据报分片。

版本：长度为4比特，用来表示IP协议的版本。通信双方使用的IP协议的版本必须一致，目前广泛使用的IP协议的版本号为4(即IPv4)
首部长度：长度为4比特，该字段的取值以4字节为单位，用来表示IPv4数据报的首部长度。最小取值为二进制的0101，即十进制的5，再乘以4字节单位，表示IPv4数据报首部只有20字节固定部分；最大取值为二进制的1111，即十进制的15，再乘以4字节单位，表示IPv4数据报首部包含20字节固定部分和最大40字节可变部分。
区分服务：长度为8比特，用来获得更好的服务，该字段在旧标准中叫做服务类型，但实际上一直没有被使用过。利用该字段的不同取值可提供不同等级的服务质量。只有在使用区分服务时才起作用，一般情况下都不使用该字段。
总长度：长度为16比特，该字段的取值以字节为单位，用来表示IPv4数据报的长度(首部长度+数据载荷长度)。最大取值为二进制的16个比特1，即十进制的65535(很少传输这么长的IPv4数据报)。
标识：长度为16比特，属于同一个IPv4数据报的各分片数据报应该具有相同的标识。IP软件会维持一个计数器，每产生一个IPv4数据报，计数器值就加1，并将此值赋给标识字段。
标志：长度为3比特。最低位(More Fragment,MF)表示本分片后面是否还有分片，MF=1表示本分片后面还有分片，MF=0表示本分片后面没有分片；中间位(Don't Fragment,DF)表示是否允许分片,DF=1表示不允许分片，DF=0表示允许分片；最高位为保留位，必须设置为0。
片偏移：长度为13比特，该字段的取值以8字节为单位，用来指出分片IPv4数据报的数据载荷偏移其在原IPv4数据报的位置有多远。
生存时间(Time TO Live,TTL)：长度为8比特，最大取值为二进制的11111111，即十进制的255，防止被错误路由的IPv4数据报无限制地在因特网中兜圈。该字段的取值最初以秒为单位。因此，IPv4数据报的最大生存时间最初为255秒，路由器转发IPv4数据报时，将其首部中该字段的值减去该数据报在路由器上所耗费的时间，若结果不为0就转发，否则就丢弃。生存时间字段后来改为以“跳数”为单位，路由器收到待转发的IPv4数据报时，将其首部中的该字段的值减1，若结果不为0就转发，否则就丢弃。
协议：长度为8比特，用来指明IPv4数据报地数据载荷是何种协议数据单元PDU。
首部检验和：长度为16比特，用于检测IPv4数据报在传输过程中其首部是否出现了差错。IPv4数据报每经过一个路由器，其首部中的某些字段的值(例如生存时间TTL、标志以及片偏移等)都可能发生变化，因此路由器都要重新计算一下首部检验和。
源IP地址：长度为32比特，用来填写发送IPv4数据报的源主机的IPv4地址
目的IP地址：长度为32比特，用来填写接收IPv4数据报的目的主机的IPv4地址

可选字段：长度从1字节到40字节不等，用来支持排错、测量以及安全措施等功能。虽然可选字段增加了IPv4数据报的功能，但这同时也使得IPv4数据报的首部长度成为可变的，这就增加了因特网中每一个路由器处理IPv4数据报的开销。但是实际上，可选字段很少被使用。
填充：用来确保IPv4数据报的首部长度是4字节的整数倍，使用全0进行填充。由于IPv4数据报的首部长度字段的值以4字节为单位，因此IPv4数据报的首部长度一定是4字节的整数倍，由于IPv4数据报手部中的可选字段的长度从1字节到40字节不等，那么当20字节的固定部分加上1到40个字节长度不等的可变部分，就可能造成IPv4数据报的首部长度不是4字节的整数倍的情况，对于这种情况，就用取值为全0的填充字段填充相应个字节，以确保IPv4数据报的首部长度是4字节的整数倍。

总长度=首部长度+数据载荷的长度
常用协议及其相应的协议字段值
ICMP      IGMP        TCP          UDP           IPv6           OSPF
1         2           6            17            41             89
```



#### IPv4数据报分片 

数据链路层帧的数据载荷长度受限于最大传送单元MTU——1500字节，如果某个IPv4数据报的总长度超过MTU，将无法封装成帧，需要将原IPv4数据报分片为若干个更小的IPv4数据报，再将各分片数据包封装成帧。



##### 分片方法

例题：某个IPv4数据报总长度为3820字节，采用20字节固定首部，根据数据链路层要求，需要该IPv4数据报分片长度不超过1420字节的数据报片。除以8的原因是片偏移字段以8字节为单位

|              | 总长度  | 标识  | MF   | DF   | 片偏移     |
| ------------ | ------- | ----- | ---- | ---- | ---------- |
| 原IPv4数据报 | 20+3800 | 23333 | 0    | 0    | 0/8=0      |
| 分片数据报1  | 20+1400 | 23333 | 1    | 0    | 0/8=0      |
| 分片数据报2  | 20+1400 | 23333 | 1    | 0    | 1400/8=175 |
| 分片数据报3  | 20+1000 | 23333 | 0    | 0    | 2800/8=350 |



##### 首部检验和的计算方法

发送端：把IPv4数据报首部划分为若干个16比特字的序列，并把首部校验和字段的值置为全0，用反码算术运算求和，把所有16比特字相加，将得到的和取反码，就得到了16比特的检验和，将其写入首部检验和字段。

接收端：接收方收到IPv4数据报后，将其首部的所有6比特字使用反码算术运算相加一次，将得到的和取反码，就得出接收方检验和的计算结果，结果为0标识首部无差错，否则表示首部出现差错。



**二进制反码求和运算过程**

两个数进行二进制反码求和的运算规则是从低位到高位逐列进行计算：
0和0相加是0
0和1相加是1
1和1相加是0，但要产生一个进位1，加到下一列
若最高位相加后产生进位，则最后得到的结果要加1



### 8.静态路由配置

静态路由配置是指用户或网络管理员使用路由器的相关命令给路由器进行人工配置路由表。人工配置方式简单、开销小，但不能及时适应网络状态(流量、拓扑等)的变化，一般只在小规模网络中采用。

当一个路由器需要向在另一个广播域内的路由器发送IP数据报，而自己的路由表里没有目的网络的路由条目。可以通过命令来给路由器添加到目的网络的路由条目。手动配置的静态路由不是路由器通过路由选择协议自动获取到的。



#### 默认路由

一个路由器想要向因特网中某个网络转发一个IP数据报，可以使用默认路由条目，替代了去往因特网中众多海量路由条目，这样极大地减少了路由表所占用的存储空间和搜索路由表所耗费的时间。将目的网络配置为0.0.0.0/0。默认路由条目中的目的网络0.0.0.0/0，0.0.0.0表示任意网络，网络前缀/0(相应的地址掩码为0.0.0.0)是最短的网络前缀。路由器在查找转发表转发IP数据报时，遵循“最长前缀匹配”的原则，因此，默认路由条目的匹配优先级最低。



#### 特定主机路由

出于某种安全问题的考虑，同时为了使网络管理员更方便地控制网络和测试网络，特别是在对网络的连接或路由表进行排错时，指明到某一台主机的特定主机路由是十分有用的。

一个路由器在自己的路由表中中为某一个网络中的某一主机进行配置，假设目的网络为192.168.2.1/32。特定主机网络条目中的目的网络192.168.2.1/32，其中192.168.2.1是特定主机的IP地址，而网络前缀“/32”(相应的地址掩码为255.255.255.255)是最长的网络前缀。路由器在查找转发表转发IP数据报时，遵循“最长前缀匹配”的原则，因此特定主机路由条目的匹配优先级最高。



#### 注意事项

进行静态路由配置需要认真考虑和谨慎操作，否则可能出现以下问题：
路由条目配置错误，可能导致出现路由环路
聚合路由条目时可能引入不存在的网络



### 9.路由选择协议

#### 路由选择分类

静态路由选择

采用人工配置的方式给路由器添加网络路由、默认路由和特定主机路由等路由条目。静态路由选择简单、开销小，但不能及时适应网络状态(流量、拓扑等)的变化。静态路由一般只在小规模网络中采用。



动态路由选择

路由器通过路由选择协议自动获取路由信息。动态路由选择比较复杂，开销比较大，但能较好地适应网络状态的变化。动态路由选择适用于大规模网络。



#### 因特网采用分层次的路由选择协议

因特网是全球最大的互联网，它采取的路由选择协议具有以下三个主要特点：
自适应：因特网采用动态路由选择。能较好地适应网络状态的变化。
分布式：因特网中的各路由器通过相互间的信息交互，共同完成路由信息的获取和更新。
分层次：将整个因特网划分为许多较小的自治系统(Autonomous System,AS)。在自治系统内部和外部采用不同类别的路由选择协议，分别进行路由选择。自治系统内部的路由选择简称为域内路由选择，自治系统之间的路由选择简称为域间路由选择。域内路由选择使用内部网络协议IGP这个类别的路由选择协议，域间路由选择使用外部网络协议EGP这个类别的路由选择协议。

在一个自治系统内部使用的具体的内部网关协议与因特网中其他自治系统中选用何种内部网关协议是无关的。

**注：**

外部网络协议EGP和内部网络协议IGP只是路由选择协议的分类名称，而不是具体的路由选择协议。

外部网络协议EGP和内部网络协议IGP名称中使用的是“网关”这个名词，是因为在因特网早期的RFC文档中，没有使用“路由器”而是使用的是“网关”这一名词。



#### 路由信息协议RIP

路由信息协议(Routing Information Protocol,RIP)是内部网关协议中最先得到广泛使用的协议之一，RIP要求自治系统AS内的每一个路由器，都要维护从它自己到AS内其他每一个网络的距离记录，这一组距离称为距离向量(Distance-Vector,D-V)。

RIP使用跳数作为度量(Metric)来衡量到达目的网络的距离。

```
RIP将路由器到直连网络的距离定义为1。
RIP将路由器到非直连网络的距离定义为所经过的路由器加1
RIP允许一条路径最多只能包含15个路由器，距离等于16时相当于不可达，因此RIP只适用于小型互联网。
```

RIP认为好的路由就是“距离短”的路由，也就是所通过路由器数量最少的路由。当到达同一目的网络有多条RIP距离相等的路由时，可以进行等价负载均衡，也就是将通信量均衡地分布到多条等价的路径上。

RIP具有以下三个重要特点：

```
和谁交换信息：仅和相邻路由器交换信息。(两个路由器直连算是相邻)
交换什么信息：路由器自己的路由表，即本路由器到所在自治系统AS中各网络的最短RIP距离，以及到各网络应经过的下一跳路由器。
何时交换信息：周期性交换(例如，间隔30s。)为了加快RIP的收敛速度，当网络拓扑发生变化时，路由器要及时向相邻路由器通告拓扑变化后的路由信息，这称为触发更新。
```



##### RIP工作过程

路由器刚开始工作时，只知道自己到直连网络的RIP距离为1；
每个路由器仅和相邻路由器周期性地交换并更新路由信息；
若干次交换和更新后，每个路由器都知道到达本自治系统AS内各网络的最短距离和下一跳路由器，称为收敛。



##### 距离向量算法

相邻路由器将自己的路由表封装在RIP更新报文中，发送给当前路由器。当前路由器收到相邻路由器发送来的路由表后，对其进行修改：将到达各目的网络的下一跳都改为相邻路由器，并且将RIP距离都加1。
当前路由器基于修改后的路由信息，来更新自己的路由表。以下是更新时可能遇到的情况：

```
1.假设自己到网络N2的RIP距离是2，下一跳为相邻路由器。下一跳经过相邻路由器的转发，现在知道了到达N2仍然要经过相邻路由器的转发，RIP距离发生了变化，自己的路由表也需要进行修改。(到达目的网络，相同的下一跳，有最新消息，需要更新)
2.假设自己原本不知道网络N3的存在，现在知道了可以通过相邻路由器到达N3。于是自己将到达N3这条路由条目添加到自己的路由表中。(发现了新的网络，需要进行添加)
3.假设自己原来到达网络N4的RIP距离是8，下一跳经过A的转发。现在知道了到达N4如果通过相邻路由器来转发，则RIP距离可缩短为5。于是，将自己到达N4的路由条目中的下一跳修改为相邻路由器，RIP距离修改为5。(到达目的网络，不同的下一跳，如果新路由RIP距离更小，新路由有优势，就需要进行更新)
4.假设自己原来到达网络N5的RIP距离是4，下一跳经过B的转发。现在知道了到达N5还可以通过相邻路由器来转发，RIP距离也为4。于是，将到达N5的这条新的路由条目，添加到自己的路由表中。( 通过不同下一跳到达目的网络，新路由的RIP距离也相同，添加新路由条目到路由表中可以进行等价负载均衡)
5.假设自己原来到达网络N6的RIP距离是4，下一跳经过C的转发。现在知道到达N6如果通过相邻路由器来转发，则RIP距离将增大到6。于是，不使用这条路由条目来更新自己的路由表。(通过不同下一跳到达目的网络，如果新路由的RIP距离增大，新路由处于劣势，不应该更新)
```

除了上述RIP路由条目更新规则，在RIP的距离向量算法中还包含以下一些时间参数：

```
路由器每隔大约30s向其所有相邻路由器发送路由更新报文。
若180s(默认)没有收到某条路由条目的更新报文，则把该路由条目标记为无效(即把RIP距离设置为16，表示不可达)；若再过一段时间(如120s)，还没有收到该路由条目的更新报文，则将该路由条目从路由表中删除。
```



##### RIP存在的问题

RIP存在的一个问题是当网络出现故障时，要经过比较长的时间才能将此信息传送到所有路由器。这一特点叫做：好消息传播得快，坏消息传播得慢。
假设路由器R1到网络N1出现了故障，R1无法到达N1，于是R1将到达N1的距离改为了16，表示不可达。但是很可能要经过30s后R1才把更新信息发送给R2，然而R2可能已经把自己的路由表发送给了R1。R1在收到R2的更新报文后，误认为可经过R2到达N1，于是根据收到的路由信息更新自己到达N1的路由信息，然后把更新后的信息发送给R2。R2在收到R1传来的信息后，更新自己的路由表，然后再次发送给R1。直到R1和R2到N1的距离都增大到16时，R1和R2才知道N1是不可达的。

”坏消息传播得慢“的问题又被称为路由环路或RIP距离无穷计数问题。这是距离向量算法的一个固有问题。可以采取以下多种措施减少出现该问题的概率或减小该问题带来的危害：

```
限制最大RIP距离为15(16表示不可达)
当路由表发生变化时就立即发送路由更新报文(即触发更新)，而不是周期性发送。
让路由器记录收到某个特定路由信息的接口，而不让同一路由信息再通过此接口反方向传送(即水平分割)
```

但是使用上述措施仍无法彻底解决问题。因为在距离向量算法中，每个路由器都缺少到目的网络整个路径的完整信息，无法判断所选的路由是否出现了环路。



##### RIP版本和相关报文的封装

较新的RIP版本是1998年11月公布的RIP2，已经成为因特网标准协议。相较RIP1，RIP2支持变长子网掩码和CIDR，还提供简单的鉴别过程并支持多播。

RIP相关报文使用运输层的用户数据报协议UDP进行封装，使用的UDP端口是520。从RIP报文封装的角度看，RIP属于TCP/IP体系结构的应用层；但RIP的核心功能是路由选择，这属于TCP/IP体系结构的网际层。



#### 开放最近路径OSPF

开放最短路径优先(Open Shortest Path First，OSPF)协议是为了克服路由信息协议RIP的缺点而开发出来的。
“开放”表明OSPF协议不是受某一厂商控制，而是公开发表的。
“最短路径优先”是因为Dijkstra提出的最短路径算法“Shortest Path First，SPF”。

**注：**”开放最短路径优先”只是一个路由选择协议，但这并不表示其他的路由选择协议不是‘最短路径优先“。实际上，用于自治系统AS内部的各种路由选择协议(例如RIP)，都要寻找一条”最短“的路径。

OSPF是基于链路状态的，而不像RIP是基于距离向量的。OSPF基于链路状态并采用最短路径算法计算路由，从算法上保证了不会产生路由环路。OSPF不限制网络规模，更新效率高，收敛速度快。



**链路状态**

链路状态(Link State，LS)是指本路由器都和哪些路由器相邻，以及相应链路的”代价(cost)“。”代价“是用来表示费用、距离、时延和带宽等，这些都是由网络管理员来决定的。例如，在思科的路由器中OSPF协议计算代价的方法是：100Mbps除以链路带宽，计算结果小于1的值仍记为1，大于1且有小数的舍去小数。



**OSPF路由器邻居关系的建立和维护**

OSPF相邻路由器分组之间通过交互问候(Hello)分组来建立和维护邻居关系。

问候(Hello)分组封装在IP数据报中，发往多播地址224.0.0.5.IP数据报首部中的协议号字段的取值为89，表明IP数据报的数据载荷为OSPF分组。

**注：**OSPF分组直接使用网络层的IP数据报进行封装，而不像RIP报文需要使用运输层用户数据协议UDP封装。从数据包按网络体系结构逐层封装的角度看，OSPF属于网络层协议，而RIP属于应用层协议(但其核心功能是路由选择，属于网络层)。

问候分组的发送周期为10s，若40s未收到来自邻居路由器的问候分组，则认为邻居路由器不可达。

每个路由器都会建立一张邻居表。邻居表里有邻居ID，连接自己的接口号，以及”判活“倒计时。



**链路状态通告**

使用OSPF的每个路由器都会产生链路状态通告(Link State Advertisement，LSA)，LSA包含以下两类链路状态信息：直连网络的链路状态信息，邻居路由器的链路状态信息。



**链路状态更新分组**

链路状态通告LSA被封装在链路状态更新(Link State Update，LSU)分组中，采用可靠的洪泛(Flooding)进行发送。洪泛法的要点是路由器向自己所有的邻居路由器发送链路状态更新分组，收到该分组的各路由器又将该分组转发给自己所有的邻居路由器(但其上有路由器除外)，以此类推。
可靠是指收到链路状态更新分组后要发送确认，收到重复的更新分组无需再次转发，但要发送一次确认。



**链路状态数据库**

使用OSPF的每一个路由器都有一个链路状态数据库(Link State DataBase，LSDB)，用于存储链路状态。通过各路由器洪泛发送封装有各自链路状态通告LSA的链路状态更新分组LSU，各路由器的链路状态数据库LSDB最终将达到一致。



**基于链路状态数据库进行最短路径优先计算**

使用OSPF的各路由器，基于链路状态数据库LSDB进行最短路径优先计算，构建出各自到达其他路由器的最短路径，即构建各自的路由表。

通过各路由器洪泛发送封装有各自链路状态通告LSA的链路状态更新分组LSU，各路由器最终会得出相同的链路状态数据库LSDB；由链路状态数据库LSDB可以得出带权有向图；对带权有向图进行基于Dijkstra的最短路径优先算法就可以得出以各路由器为根的最短路径。



##### OSPF的五种分组类型

问候(Hello)：用来发现和维护邻居路由器的可达性。
数据库描述(Database Description)：用来向邻居路由器给出自己的链路状态数据库中所有链路状态项目的摘要信息。
链路状态请求(Link State Request)：用来向邻居路由器请求发送某些链路状态项目的详细信息。
链路状态更新(Link State Update)：路由器使用链路状态更新分组将其链路状态信息进行洪泛发送，即用洪泛法对整个系统更新链路状态。
链路状态确认(Link State Acknowledgement)：对链路状态更新分组的确认分组。



##### OSPF的基本工作过程

相邻路由器R1和R2之间，每隔10s要交换一次问候分组，以便建立和维护邻居关系。建立邻居关系后，会给邻居路由器发送数据库描述分组，也就是将自己的链路状态数据库中的所有链路状态项目的摘要信息发送给邻居路由器。假设路由器R1收到R2的数据库描述分组后，发现自己缺少其中的某些链路状态项目，于是就给R2发送链路状态请求分组。R2收到R1发来的链路状态请求分组后，将R1所缺少的链路状态项目的详细信息封装在链路状态更新分组中发送给R1。R1收到R2发来的链路状态更新分组后，将这些缺少的链路状态项目的详细信息添加到自己的链路状态数据库中，并给R2发送链路状态确认分组，R2也可以向R1请求，自己所缺少的链路状态项目的详细信息。R1和R2的链路状态数据库最终将达到一致，即同步。

两个同步的路由器称为完全邻接的路由器。

每30min或链路状态发生变化时，路由器都会洪泛发送链路状态更新分组，收到该分组的其他路由器会根据分组的内容更新自己的链路状态数据库，然后洪泛转发该分组，并给该路由器发回链路状态确认分组，这又称为新情况下的链路状态数据库同步



##### 多点接入网络中的OSPF路由器

假设现在有5台路由器连接在同一个多点接入网络中，它们每隔10s发送一次问候分组，以建立和维护邻居关系。这些路由器中的任意两个路由器都互为邻居关系，邻居关系的数量为[n*(n-1)/2，其中n是路由器的数量]。因此，每个路由器需要向其他(n-1)个路由器发送问候分组和链路状态更新分组。

为了减少所发送问候分组和链路状态更新分组的数量，OSPF采用选举指定路由器(Designated Router，DR)和备用的指定路由器(Backup Designated Router，BDR)的方法。所有的非DR/BDR只与DR/BDR建立邻居关系，因此邻居关系数量降为[2*(n-2)+1]。非DR/BDR之间不能直接交换信息，而必须通过DR/BDR交换信息。若DR出现问题，则由BDR顶替DR。实现DR和BDR的选举是通过各路由器之间交换一些选举参数(例如路由器优先级，路由器ID以及接口IP地址等)，然后根据一定的选举规则选举出DR和BDR，这与以太网交换机生成树协议选举根交换机类似。



##### OSPF划分区域

为了使OSPF协议能够用于规模很大的网络，OSPF把一个自治系统AS再划分为若干个更小的范围，称为区域(area)。每个区域都有一个32比特的区域标识符，可以用点分十进制表示。主干区域的标识符必须为0，也可表示成点分十进制形式的0.0.0.0。主干区域用于连通其他区域，其他区域的标识符不能为0且互不相同。每个区域的规模不应过大，一般所包含的路由器不应超过200个。

划分区域的好处：把利用洪泛法交换链路状态信息的范围局限于每一个区域，而不是整个自治系统AS，这样就减少了整个网络上的通信量。

在主干区域，还要有一个路由器专门和本自治系统外的其他自治系统交换路由信息，这样的路由器称为自治系统边界路由器(AS Border Router，ASBR)。主干区域内的路由器称为主干路由器(Backbone Router，BBR)。如果路由器的所有接口都在同一个区域内，则该路由器称为区域内路由器(Internal Router，IR)。为了本区域可以和自治系统内的其他区域连通，每个区域都会有一个区域边界路由器(Area Border Router，ABR)，区域边界路由器的一个接口用于连接自身所在区域，另一个接口用于连接主干区域。

采用划分区域的方法，虽然使交换信息的种类增多了，同时也使OSPF协议更加复杂了，但这样做能使每一个区域内部交换路由信息的通信量大大减小，因而使OSPF协议能够用于规模更大的自治系统AS。



#### 边界网关协议BGP

边界网关协议(Border Gateway Protocol，BGP)属于外部网关协议EGP这个类别，用于自治系统AS之间的路由选择协议。由于在不同的AS内度量路由的”代价“(距离、带宽、费用等)可能不同，因此对于AS之间的路由选择，使用统一的"代价"作为度量来寻找最佳路由时不行的。

BGP只能是力求寻找一条能够到达目的网络且比较好的路由(即不能兜圈子)，而并非要寻找一条最佳路由。

在配置BGP时，每个AS的管理员要选择至少一个路由作为该AS的"BGP发言人"。一般来说，两个BGP发言人都是通过一个共享网络连接在一起，而BGP发言人往往就是BGP边界路由器。不同自治系统的BGP发言人要交换路由信息，首先必须建立TCP连接，使用端口179；在此TCP连接上交换BGP报文，以建立BGP会话；利用BGP会话交换路由信息。使用TCP连接交换路由信息的两个BGP发言人彼此称为对方的邻站(neighbor)或对等站(peer)。BGP发言人除了运行BGP协议外，还必须运行自己所在AS所使用的内部网关协议IGP，例如RIP或者OSPF。

BGP发言人交换网络可达性的信息，也就是要到达某个网络锁要经过的一系列自治系统。当BGP发言人相互交换了网络可达性的信息后，各BGP发言人就根据所采用的策略，从收到的路由信息中找出到达各自治系统的较好地路由，也就是构造出树形结构且不存在环路的自治系统连通图。

BGP适用于多级结构的因特网。(AS1,AS2)这种交换信息时使用的路径称为路径向量，若AS3的BGP发言人收到这条路径向量信息后，发现AS3自身也包含在其中，则不能采用这条路径，否则会兜圈子。



##### BGP-4的四种报文

BGP-4是目前使用的最多的版本，RFC4271规定了其四种报文：
打开(Open):用来与相邻的另一个BGP发言人建立关系，是通信初始化。
保活(Keepalive):用来周期性地证实邻站的连通性。
更新(Update):用来通告某一条路由的信息，以及列出要撤销的多条路由。
通知(Notification):用来发送检测到的差错。



### 10.路由器的基本工作原理

路由器是一种具有多个输入端口和输出端口的专用计算机，任务是转发分组。

路由器由路由选择部分和分组转发部分这两部分构成。路由选择部分的核心构件时路由选择处理机，其任务是根据所使用的路由选择协议周期性地与其他路由器进行路由信息的交换，以便构建和更新路由表。

分组转发部分由交换结构、一组输入端口以及一组输出端口构成。信号从路由器的某个输入端口进入路由器，物理层将信号转换成比特流后，向上交付给数据链路层处理；数据链路层从比特流中识别出帧，去掉帧头和帧尾后，向上交付给网络层处理。

如果交付给网络层处理的分组是普通待转发的数据分组，则根据分组首部中的目的地址进行查表转发，若找不到匹配的转发条目，就丢弃该分组，否则按照匹配条目中所指示的端口进行转发。网络层更新数据分组首部中的某些字段的值，例如将数据分组的生存时间TTL减1，然后向下交付给数据链路层进行封装；数据链路层将数据分组封装成帧后，向下交付给物理层处理；物理层将帧看作比特流，将其变换成相应的电信号发送出去。

如果交付给网络层处理的是携带有路由报文的路由分组，则把路由保温送交路由选择处理机。路由选择处理机根据路由报文的内容来更新自己的路由表，路由表一般仅包含从目的网络到下一跳的映射。路由表需要对网络拓扑变化的计算最优化。而转发表是从路由表得出来的。转发表的结构应当使查找过程最优化。

路由选择处理机除了处理收到的路由报文，还会周期性地给其他路由器发送自己所知道的路由信息。路由器的各端口还应具有输入缓冲区和输出缓冲区，输入缓冲区用来暂存新进入路由器但还来不及处理的分组，输出缓冲区用来暂存已经处理完毕但还来不及发送的分组。

交换结构的速率对于路由器的性能是至关重要的。因此，人们对交换结构进行了大量研究，以提高路由器的转发速率。实现交换结构的三种基本方法是：通过存储器，通过总线以及通过互连网络。这三种交换结构可实现的路由器转发速率依次提高。



### 11.网际控制报文协议ICMP

为了更有效的转发IP数据报以及提高IP数据报交付成功的机会，TCP/IP体系结构的网际层使用了网际控制报文协议(Internet Control Message Protocol，ICMP)。主机或路由器使用ICMP来发送差错报告报文和询问报文。ICMP报文被封装在IP数据报中发送。它作为IP数据报的数据载荷，IP协议为其添加一个IP首部，使之成为IP数据报。

```
ICMP报文格式
0            8           16                         31
类型         代码         检验和
这四个字节取决于ICMP报文的类型
ICMP报文的数据载荷(长度取决于类型)
```



#### ICMP报文类型

ICMP报文分为以下两大类：
差错报告报文：用来向主机或路由器报告差错情况
询问报文：用来向主机或路由器询问情况



##### 差错报告报文

常见的ICMP差错报告报文有以下五种：
终点不可达：当路由器或主机不能交付IP数据报时，就向源点发送终点不可达报文。具体可再根据ICMP的代码字段细分为目的网络不可达、目的主机不可达、目的协议不可达、目的端口不可达、目的网络未知、目的主机未知等13种。
源点抑制：当路由器或主机由于拥塞而丢弃IP数据报时，就向发送该IP数据报的源点发送远点抑制报文，使源点直到应当把IP数据报的发送速率放慢。
时间超过(超时)：当路由器收到一个目的IP地址不是自己的IP数据报时，会将其首部中的生存时间TTL字段的值减1。若结果不为0，则路由器将该数据报转发出去；若结果为0，路由器不但要丢弃该数据报，还要向该IP数据报的源点发送时间超过(超时)报文。另外，当终点在预先规定的时间内未能收到一个数据报的全部数据报分片时，就把已收到的数据报片都丢弃，也会向源点发送时间超过(超时)报文。
参数问题：当路由器或目的主机收到IP数据报后，发现其首部中有不正确的字段值时，就丢弃该数据报，并向发送该数据报的源点发送参数问题报文。
改变路由(重定向)：路由器把改变路由报文发送给主机，让主机知道下次应将IP数据报发送给另外的路由器，这样可以通过更好的路由达到目的主机

**注：以下情况不发送ICMP差错报告报文**

```
1.对ICMP差错报告报文不再发送ICMP差错报告报文
2.对第一个分片的IP数据报片的所有后续数据报片都不发送ICMP差错报告报文
3.对具有多播地址的IP数据报都不发送ICMP差错报告报文
4.对具有特殊地址(例如127.0.0.1或0.0.0.0)的IP数据报不发送ICMP差错报告报文
```



##### 询问报文

常用的ICMP询问报文有以下两种：
回送请求和回答：由主机或路由器向一个特定的目的主机或路由器发出。收到此报文的主机或路由器必须给发送该报文的源主机或路由器发送ICMP回送回答报文。这种询问报文用来测试目的站是否可达以及了解其有关状态。
时间戳请求和回答：用来请求某个主机或路由器回答当前的日期和时间。在ICMP时间戳回答报文中有一个32比特的字段，其中写入的整数代表从1900年1月1日起到当前时刻一共有多少秒。这种询问报文用来进行时钟同步和测量时间。



#### ICMP典型应用

##### 分组网间探测

分组网间探测(Packet InterNet Groper，PING)PING用来测试主机或路由器之间的连通性。该应用是TCP/IP体系结构的应用层直接使用网际层ICMP的一个例子，它并不使用运输层的TCP或者UDP。PING应用所使用的ICMP报文类型是回送请求和回答。某些主机或服务器为了防止恶意攻击，并不会理睬外界发来的ICMP回送请求报文。

跟踪路由应用traceroute，用于探测IP数据报从源主机到达目的主机要经过那些路由器。在不同的操作系统中，traceroute应用的命令和实现机制有所不同：

在Windows版本中，具体命令为"tracert"，其应用层直接使用网际层的ICMP协议，所使用的ICMP报文类型由回送请求和回答报文以及差错报告报文。

### 12.虚拟专用网VPN和网络地址转换NAT

#### 虚拟专用网VPN

虚拟专用网(Virtual Private Network，VPN)：利用公用的因特网作为本机构各专用网之间的通信载体，这样形成的网络又称为虚拟专用网。出于安全考虑，专用网内的各主机并不应该直接“暴露”于公用的因特网上。因此，给专用网内各主机配置的IP地址应使各主机在专用网内可以相互通信，而不能直接与公用的因特网通信。给专用网内各主机配置的IP地址应该是该专用网所在机构可以自行分配的IP地址，这类IP地址仅在机构内部有效，称为专用地址(Private Address)，不需要向因特网的管理机构申请。

RFC 1918规定了以下三个CIDR地址块中的地址作为专用地址：

```
10.0.0.0~10.255.255.255 (CIDR地址块 10/8)
172.16.0.0~172.31.255.255 (CIDR地址块 172.16/12)
192.168.0.0~192.168.255.255 (CIDR地址块 192.168/16)
```

因特网中的所有路由器，对目的地址是专用地址的IP数据报一律不进行转发，这需要由因特网服务提供者ISP对其拥有的因特网路由器进行设置来实现。



##### 使用VPN进行发送数据的过程

```
两个专用网中的主机都配置好了相应的专用地址，每一个专用网中都需要有一个具有合法的全球IP地址的路由器，这样两个专用网才能利用公用的因特网进行通信。另外，还需要在这两个路由器上进行相应的VPN配置。

当一个专用网中的主机A要向另一个专用网中的主机B发送数据时，A将待发送数据封装成内部IP数据报发送给路由器R1，内部IP数据报首部中源地址字段的值为A的IP地址，目的地址为B的IP地址。

R1收到该内部IP数据报后，发现其目的网络必须通过因特网才能到达，就将该内部IP地址数据报进行加密，这样就确保了内部IP数据报的安全，然后重新填加上一个首部，封装成可在因特网上传送的外部数据报，外部数据报首部中源地址字段的值为路由器R1的全球唯一的公有IP地址，目的地址字段的值为路由器R2的全球唯一的公有IP地址。

R1将该外部数据报通过因特网发送给R2，R2收到该外部数据报后，去掉其首部，将其数据载荷，即加密后的内部IP数据报进行解密，恢复出原来的内部IP数据报。这样就可以从内部IP数据报首部提取出源地址和目的地址，并根据目的地址将该内部IP数据报发送给目的主机B。
```

虽然两个专用网内的主机间发送数据是通过专用网的因特网传送的，但从效果上就好像是本机构的专用网上传送一样，这就是虚拟专用网中“虚拟”的含义。

IP数据报在因特网中可能要经过多个网络和路由器，但逻辑上看，路由器R1和R2之间好像是一条直通的点对点链路，因此也被称为IP隧道技术。

在同一机构内不同部门的内部网络所构成的VPN被称为**内联网VPN**。一个机构的虚拟专用网需要某些外部机构参加进来，这样的VPN称为**外联网VPN**。在外地工作的员工需要访问公司内部的专用网时，只要在任何地点接入因特网，运行驻留员工PC中的VPN软件，在员工的PC和公司的主机之间建立VPN隧道，就可以访问专用网中的资源，这种VPN称为**远程接入VPN**。



#### 网络地址转换NAT

网络地址转换(Network Address Translation，NAT)是1994年提出，用来缓解IPv4地址空间即将耗尽的问题(2011年，IPv4地址已经分配完毕)。NAT能使大量使用内部专用地址的专用网络用户共享少量外部全球地址来访问因特网上的主机和资源。这种方法需要在专用网络连接到因特网的路由器上安装NAT软件。装有NAT软件的路由器称为NAT路由器，它至少要有一个有效的外部全球地址IPG。这样，所有使用内部专用地址的主机在和外部因特网通信时，都要在NAT路由器上将其内部专用地址转换成IPG。



##### 最基本的NAT方法

```
主机A位于专用网中，其IP地址为专用地址IPA；主机B位于因特网中，其IP地址为全球地址IPB；A所在的专用网连接因特网的路由器为NAT路由器，它至少拥有一个有效的外部全球IP地址IPG。

主机A给主机B发送一个IP数据报，该数据报首部中的源地址为主机A的专用地址IPA，目的地址为主机B的全球地址IPB，该数据报必须经过NAT路由器的转发，NAT路由器收到该数据报后，从自己的全球地址池中，为主机A选择一个临时的全球地址IPG，将数据报首部中的源地址修改为IPG，并在自己的NAT转换表中记录IPA与IPG的对应关系。之后，将修改过源地址的数据报转发到因特网。

主机B收到该IP数据报后，给主机A发回一个IP数据报，该IP数据报首部中的源地址为主机B的全球地址IPB，目的地址为NAT路由器为主机A选择的临时全球地址IPG。当NAT路由器从因特网上收到该数据报时，根据NAT转换表中的相关记录，NAT路由器会将该数据报转发给专用网中的主机A。因此，NAT路由器将数据报首部中的目的地址修改为主机A的专用地址IPA后，转发给专用网中的主机A。
```

缺点：如果NAT路由器拥有n(n比较小)个全球IP地址，那么专用网内最多可以同时有n台主机接入因特网。若专用网内的主机数量大于n，则需要轮流使用NAT路由器中数量较少的全球IP地址。



##### 网络地址与端口号转换方法

由于目前绝大多数基于TCP/IP协议栈的网络应用，都是用运输层的传输控制协议TCP或用户数据报协议UDP，为了更加有效地利用NAT路由器中的全球IP地址，现在常将NAT转换和运输层端口号结合使用。这样就可以使内部专用网中使用专用地址的大量主机，共用NAT路由器上的1个全球IP地址，因而可以同时与因特网中的不同主机进行通信。

将NAT和运输层端口号结合使用，称为网络地址与端口号转换(Network Address and Port Translation，NAPT)。现在很多家用路由器将家中各种只能设备()

## 八、无线网络

### 1.802.11 无线局域网的组成

IEEE于1997年制定了无线局域网(Wireless Local Area Network，WLAN)的协议标准802.11，802.11无线局域网是目前应用最广泛的无线局域网之一，人们更多地将其简称为Wi-Fi(Wireless Fidelity，无线保真度)。

802.11无线局域网可分为以下两类：有固定基础设施的和无固定基础设施的。固定基础设施是指预先建立的，能够覆盖一定地理范围的、多个固定的通信基站。



#### 有固定基础设施的802.11无线局域网

常采用星型网络拓扑，位于其中心的基站被称为接入点，缩写成AP。802.11无线局域网的最小构件成为基本服务集，缩写成BSS。在一个BSS中包含有一个AP和若干个移动站，本BSS内各站点之间的通信以及与本BSS外的站点之间的通信，都必须经过本BSS内的AP进行转发。网络管理员需要为AP分配一个最大32字节的服务集标识(Service Set Identifier，SSID)和一个无线通信信道，SSID实际上就是使用该AP的802.11无线局域网的名字。一个BSS所覆盖的地理范围称为基本服务区，缩写为BSA。一个BSA直径不超过100m。一个BSS可以是孤立的，也可以通过一个分配系统(DS)与其他BSS相连，这样就构成了一个扩展的服务集(ESS)。DS最常用的是以太网，也可以使用点对点链路或其他无线网络。

ESS还可以为无线用户提供到其他非802.11无线局域网的接入，如通过DS有线连接到因特网。一个BSS中的移动站A想要发送数据到另一个BSS中的移动站B，必须经过AP1，DS，AP2，最终将数据转发到移动站B。若移动站A从BSS1漫游到BSS2，在漫游过程中，移动站A的接入点从BSS1中的AP1改为了BSS2中的AP2，但移动站A仍然可保持与BSS2中移动站B的通信。(漫游是指无线网络[Wi-Fi/蜂窝网]中的一个终端设备再不同接入点之间切换的过程)



802.11标准没有定义实现漫游的具体方法，仅定义了基本服务：

```
关联服务：
移动站与接入点AP建立关联的方法有以下两种：
被动扫描：AP会周期性发出信标帧。信标帧中包含服务集标识符SSID，AP的MAC地址，所支持的速率、加密算法和安全配置等参数。移动站被动等待接收信标帧。

主动扫描：
移动站主动发出探测请求帧，然后等待来自AP的探测响应帧。

重建关联服务和分离服务：
如果一个移动站要把某个接入点AP的关联转移到另一个AP，就可以使用重建关联服务；若要终止关联服务，就应使用分离服务。
```



#### 无固定基础设施的802.11无线局域网

无固定基础设施802.11无线局域网也称为自组织网络。自组织网络并没有预先建立的固定的基础设施，如基站或接入点AP，它是由一些对等的移动站点构成的临时网络，数据在自组织网络中被多跳存储转发。转发站需要具备路由功能，自组织网络有其特定的路由选择协议，一般不能和因特网直接相连，需要通过网关或协议转换器接入到因特网。

自组织网络组网方便，不需要基站，并且具有非常好的生存性，这使得自组织网络再军用和民用领域都有很好的应用前景。802.11无线局域网的ad hoc模式允许网络中的各站点在其通信范围内直接通信，也就是支持站点间单跳通信，而标准中并没有报扩多跳路由功能。因此，802.11无线局域网的ad hoc模式应用较少。



### 2.802.11无线局域网使用CSMA/CA协议的原因

对于802.11无线局域网，其使用的无线信道传输数据，这与共享总线以太网有线传输介质不同。因此，802.11无线局域网不能简单照搬共享总线以太网使用的CSMA/CD协议。802.11无线局域网采用了载波监听多址接入/碰撞避免(Carrier Sense Mutiple Access/Collision Avoidance，CSMA/CA)。CSMA/CA协议仍然采用CSMA/CD协议中的CSMA，以“先听后说”的方式来减少碰撞的发生，但是将“碰撞检测CD”改为了“碰撞避免CA”，CA表示碰撞避免，但并不能避免所有的碰撞，而是尽量减少碰撞发生的概率。



#### 不采用碰撞检测CD的原因：

由于无线信道的传输环境复杂且信号强度的动态范围非常大，在802.11无线网卡上接收到的信号强度一般都远远小于发送信号的强度，信号强度甚至相差百万倍。因此，如果要在802.11无线网卡上实现碰撞检测，对硬件的要求非常高。

即使能够在硬件上实现碰撞检测功能，但由于无线电波传播的特殊性(存在隐蔽站问题)，还会出现无法检测到碰撞的情况，因此实现碰撞检测并没有意义。



##### 无线局域网的隐蔽站问题

移动站A无法检测到移动站B发送的信号，移动站B也无法检测到移动站A发送的信号。在A和B同时向接入点AP发送数据，必然会产生碰撞。然而，根据无线信道的信号传播特点，碰撞信号的强度比较弱，即使分别转播到了A和B，A和B也无法检测到极度微弱的碰撞信号。



### 3.CSMA/CA协议的基本工作原理

#### 相关概念

```
分布协调功能DCF(DIstributed Coordination Function，DCF)。分布协调功能帧间间隔(DIstributed Coordination Function IFS，DIFS)。DCF帧间间隔DIFS的长度为128us，在DCF方式中，DIFS用来发送数据帧和管理帧。DCF不采用任何中心控制，而是在每一个节点使用CSMA机制的分布式接入算法，让各个站通过争用信道来获取发送权。

短帧间间隔(Short Inter-Frame Spacing，SIFS)，长度为28us，它是最短的帧间间隔，用来分隔开属于一次对话的各帧。一个站点应当能够在这段时间内从发送方式切换到接收方式，使用SIFS的帧类型有ACK帧、CTS帧等。

网络分配向量(Network Allocation Vector，NAV)指出了信道忙的持续时间/完成这次帧的传输且信道转入空闲状态所需要的时间。

请求发送RTS(Request To Send)帧是短的控制帧，包括源地址、目的地址和本次通信(包括目的站发回确认帧所需时间)所需的持续的时间。

允许发送CTS(Clear To Send)帧是短的响应控制帧，它也是包括本次通信所需的持续时间(从RTS帧中将此持续时间复制到CTS帧中)
```

​	假设一开始信道是空闲的，源站要发送它的第一个数据帧，若检测到信道空闲，则在等待间隔后才能发送。等待DIFS间隔是考虑到可能有其他的站有高优先级的帧要发送。

为避免碰撞，802.11标准规定了每个站必须同时使用以下两个方法：

```
1.用软件实现虚拟载波监听：
让源站把要占用信道的时间(DATA+SIFS+ACK)，以us为单位，写入其数据帧DATA的首部。

2.物理层用硬件实现载波监听：
每个站检查收到的信号强度是否超过一定的门限数值，用此判断是否有其他移动站在信道上发送数据。只要监听到信道忙，就不能发送数据。
```

目的站若正确收到该帧，在经过SIFS间隔后，向源站发送确认帧ACK。若源站在重传计时器设置的超时时间内没有收到ACK，就必须重传之前已发送的数据帧，直到收到ACK为止；或者经过若干次的重传失败后，放弃发送。

当某个站检测到正在信道中传输的帧首部的持续时间字段时，就调整自己的网络分配向量NAV。

目的站在收到正确的数据帧后，需要给源站发送确认帧，因为无线信道的误码率较高，CSMA/CA协议还需要使用停止等待协议的确认机制来实现可靠传输。

在NAV这段时间内若其他站也有帧要发送，就必须推迟发送，在NAV这段时间结束后，再经过一个DIFS间隔，然后再退避一段随机时间后，才能发送帧。退避原因是：当某个站在发送帧时，很有可能有多个站都在监听信道并等待发送帧，一旦信道空闲，这些站几乎同时发送帧而产生碰撞。为了避免碰撞，所有要发送帧的站检测到信道从忙转为空闲后，都要执行退避算法，这样不仅可以减少发生碰撞的概率，还可以避免某个站长时间占用无线信道。

当某个站要发送数据帧时，仅在这种情况下才不使用退避算法：检测到信道空闲，并且该数据帧不是成功发送完上一个数据帧之后立即连续发送的数据帧。除此以外的以下情况都必须使用退避算法：在发送帧之前检测到信道处于忙态；在每一次重传一个帧时；在每一次成功发送帧后要连续发送下一个帧时。



#### 退避算法

在执行退避算法时，站点为退避计时器设置一个随机的退避时间：当退避计时器的时间减小到零时，就开始发送数据；当退避计时器的时间还未减小到零时而信道又转变为忙状态，这时就冻结退避计时器的数值，重新等待信道变为空闲，再经过帧间间隔DIFS后，继续启动退避计时器。在进行第i次退避时，退避时间在时隙编号{0,1,2,...,(2^4+i - 1)}中随机选择一个，然后乘以基本退避时间(也就是一个时隙的长度)就可以得到随机的退避时间，当时隙编号达到255时(对应于第6次退避)就不再增加了。



#### 对信道进行预约

为进一步降低发生碰撞的概率，802.11无线局域网允许源站对信道进行预约

源站想要发送第一个数据帧，若检测到信道空闲，则在等待DIFS帧间间隔后，先发送一个请求发送RTS(Request To Send)帧，若目的站正确收到源站发来的RTS帧，在检测到信道空闲且经过帧间间隔SIFS后，就向源站发送一个允许发送CTS(Clear To Send)帧作为响应。源站正确收到CTS帧后，在等待一个帧间间隔SIFS后就可发送第一个数据帧了。目的站若正确收到该帧，则经过SIFS间隔后，向原站发送确认帧ACK。

除源站和目的站的其他各站，在收到CTS帧或数据帧后就推迟访问信道，这样就确保了源站和目的站之间的通信不会受到其他站的干扰，若RTS帧发生碰撞。源站就不可能收到CTS帧，源站会执行退避算法重传RTS帧。

使用RTS帧和CTS帧进行信道预约会带来额外的开销，但由于RTS帧和CTS帧都很短，发生碰撞的概率、碰撞产生的开销以及本身的开销都很小，对于一般的数据帧，其发送时延往往远大于传播时延(因为是局域网)，碰撞的概率很大，且一旦发生碰撞而导致数据帧重发，则浪费的时间就很多，因此用很小的代价对信道进行预约往往是值得的。

尽管如此，802.11无线局域网仍为用户提供了以下选择：

```
使用RTS帧和CTS帧
只有当数据帧长超过某个数值时才使用RTS帧和CTS帧
不适用RTS帧和CTS帧。
```

由于RTS帧和CTS帧都会携带通信需要持续的时间，这与前面数据帧可以携带通信所需持续时间的虚拟载波监听机制是一样的，因此，使用RTS帧和CTS帧进行信道预约，也属于虚拟载波监听机制。利用虚拟载波监听机制，站点只要监听到数据帧、RTS帧或CTS帧中的任何一个，就能知道信道将被占用的持续时间，而不需要真正的监听到信道上的信号，因此虚拟载波监听机制能减少隐藏站带来的碰撞问题。



### 4.802.11无线局域网的MAC帧

802.11无线局域网MAC帧分为三种类型：数据帧，控制帧和管理帧。

数据帧用于在站点之间传输数据。

控制帧通常与数据帧搭配使用，负责区域的清空、虚拟载波监听的维护以及信道的接入，并于收到数据帧时予以确认。ACK帧，RTS帧和CTS帧等都属于控制帧。

管理帧用于加入或退出无线网络，以及处理AP之间连接的转移事宜。信标帧、关联请求帧以及身份确认帧等都属于管理帧。
